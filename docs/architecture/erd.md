# 데이터베이스 설계 (ERD)

본 문서는 SUDA 프로젝트 MVP 버전의 데이터베이스 아키텍처를 정의합니다.

## 1. MVP 아키텍처: 무상태(Stateless) 기반 API Gateway

현재 MVP 단계에서는 사용자의 접근성을 극대화하기 위해 **"로그인 불필요"** 및 개인정보 보호를 위한 **"데이터 완전 비저장"** 원칙을 적용합니다.

따라서, 별도의 관계형 데이터베이스(PostgreSQL)를 구축하지 않는 **무상태(Stateless) 아키텍처**를 채택합니다.

### 1.1. 설계 핵심 원칙
- **DB-Free:** 회원, 프로필, 세션, 대화 기록 등 어떠한 데이터도 디스크(DB)에 영구 보관하지 않습니다.
- **Pure API Gateway:** Spring Boot 서버는 안드로이드 클라이언트와 외부 AI(Gemini, Clova) 사이에서 프롬프트를 조합하고 데이터를 중계하는 역할만 수행합니다.
- **휘발성 메모리:** 추후 API 호출 빈도 제한(Rate Limiting)이나 임시 상태 관리가 필요할 경우, 메모리 기반의 Redis만을 제한적으로 도입합니다.

## 2. 향후 고도화 계획 (Future Plan)
정식 버전 배포 시 '학습 모드' 및 '회원 관리' 기능이 추가됨에 따라 아래와 같은 도메인이 추가될 예정입니다.
- `members`, `child_profiles` (회원 및 자녀 프로필)
- `learning_sessions`, `learning_words` (수어/발음 학습 진도 관리)
