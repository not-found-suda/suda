## 주요 변경 사항
- **`SignRecognitionScreen` 리팩토링**: 중복된 권한 체크 로직을 제거하고 카메라 프리뷰 및 프레임 분석 전용 컴포넌트로 단순화
- **`FakeSignRecognitionEngine` 구현**: A파트 엔진 완성 전 UI/통신 테스트를 위해 1~2초 간격 가짜 수어 이벤트를 방출하는 테스트 엔진 구축
- **`ConversationViewModel` 도입**: 세션 상태(Idle/Active) 관리 및 엔진 이벤트 구독을 통한 실시간 데이터 흐름 제어 로직 구현
- **`ConversationRoute` UI 개발**: 상단 세션 상태 표시, 중앙 카메라 영역, 하단 인식 결과(Chip 형식) 및 시작/종료 버튼 구현
- **Hilt DI 설정**: 인터페이스 기반의 `SignRecognitionEngine` 바인딩을 통해 향후 실제 엔진으로의 손쉬운 교체 구조 확립
