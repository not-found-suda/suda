## Description

## 요약
- 퀴즈 현재 문제 응답의 targetText를 사용하도록 변경하고, 퀴즈 시작 전 words 선조회 의존을 제거

## 작업 배경
- 관련 이슈: #S14P31A404-302
- 참고 문서/기획:
  - 백엔드 `GET /api/v1/learn/quizzes/sessions/{sessionId}/questions/current` 응답에 `targetText` 추가
  - 기존 모바일은 퀴즈 시작 전 `GET /learn/words`를 호출해 `wordId -> word` 매핑을 생성

## 주요 변경 사항
- `QuizQuestionViewModel`에서 `LearningWordRepository` 의존성 제거
- 퀴즈 시작 흐름을 `activeChildId 조회 -> 세션 생성 -> 현재 문제 조회`로 단순화
- 현재 문제의 단어 표시 기준을 `current question` 응답의 `targetText`로 통일
- `targetText` 누락/빈 값인 경우 기존 오류 상태로 방어 처리
- 변경하지 않은 것 / 스코프 아웃: 단어장 화면의 words 조회 로직은 유지

## 스크린샷/로그
- UI 변경 없음
- 퀴즈 시작 전 불필요한 `/learn/words` 호출 제거
