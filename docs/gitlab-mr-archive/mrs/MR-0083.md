```text
**\[P1\] 변경 가능한 migration 태그로 인해 다른 커밋이 실행될 수 있음**

현재 migration 이미지를 develop/latest 같은 변경 가능한 태그와 CI_COMMIT_SHORT_SHA 태그로 같이 push하고 있는데, Jenkins webhook에는 어떤 커밋 태그를 실행해야 하는지가 전달되지 않는 것 같아요.

그래서 Jenkins가 이미지를 pull하기 전에 다른 pipeline이 같은 develop/latest 태그를 다시 push하면, 의도했던 커밋이 아니라 다른 커밋의 migration SQL이 실행될 가능성이 있습니다. 안정성을 위해 Jenkins에 CI_COMMIT_SHORT_SHA 같은 고정 태그를 전달해서 해당 이미지로 migration을 실행하도록 맞추면 좋을 것 같습니다!
```
