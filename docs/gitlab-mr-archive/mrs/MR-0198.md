## 요약

`- 퀴즈 세션 생성 시 문제 선조회 및 무음(STT 실패) 답변 처리 개선`

---

## 작업 배경

* 관련 이슈: S14P31A404-417
* 참고 문서/기획: 모바일 퀴즈 UX 개선 요구사항 (무음 시 다음 문제 진행 불가 이슈)

기존 구조에서는

* 한 문제씩 조회하는 방식이었고
* STT 실패(무음 포함) 시 예외 발생 → 답변 저장 실패 → 다음 문제 진행 불가

→ 모바일에서 실패 화면은 표시 가능하지만 **세션 흐름이 끊기는 문제 발생**

---

## 주요 변경 사항

* 세션 생성 시 문제 5개를 한 번에 응답하도록 prefetch 구조로 변경
* QuizSessionCreateResponse에 `questions` 리스트 추가
* audioFile 없이도 답변 제출 가능하도록 API 수정 (`required=false`)
* STT 실패/무음 시 예외 대신 `recognizedText=""`로 처리
* 무음 답변도 1점으로 저장하여 다음 문제로 정상 진행 가능하도록 개선

---

## 스크린샷/로그

* 세션 생성 응답

```json
{
  "sessionId": 10,
  "questions": [
    {
      "questionId": 1,
      "questionNumber": 1,
      "targetText": "사과"
    }
  ]
}
```

* 무음 답변 제출 응답

```json
{
  "recognizedText": "",
  "isCorrect": false,
  "star": 1,
  "hasNext": true,
  "nextQuestionNumber": 2
}
```
