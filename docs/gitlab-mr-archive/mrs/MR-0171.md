### nej0720 at 2026-05-14T17:14:52.307+09:00 (comment)

```text
\[P2\] 같은 세션에 동시 저장 요청이 오면 messageOrder가 충돌할 수 있습니다

현재 메시지를 저장할 때 `session.getMessageCount() + 1`로 다음 `messageOrder`를 만들고 있는데, 같은 대화 세션에서 부모 수어 변환과 아이 음성 변환 요청이 거의 동시에 들어오면 두 트랜잭션이 같은 `messageCount`를 읽어서 동일한 `messageOrder`로 저장을 시도할 수 있을 것 같습니다. DB에는 `(session_id, message_order)` unique 제약도 있어서 이 경우 한 요청이 실패하고, 번역 자체는 성공했는데 대화 저장/응답이 500으로 떨어질 수 있습니다.

세션 row를 pessimistic lock으로 조회해서 순번 증가를 직렬화하거나, message order 생성을 DB/락 기반으로 처리하면 더 안전할 것 같습니다.
```
