## Description

## 요약

* 퀴즈 결과 API에 세션 메타데이터를 추가하고 완료되지 않은 세션의 결과 조회를 제한하도록 수정

## 작업 배경

* 관련 이슈: #S14P31A404-363
* 참고 문서/기획:

  * 모바일 퀴즈 결과 화면 "다시 풀기" 기능 연동 대응
  * 퀴즈 완료 전 결과 조회 정책 명세 반영

## 주요 변경 사항

* 퀴즈 결과 조회 API에서 완료되지 않은 세션 조회 시 `409 LEARN_QUIZ_NOT_COMPLETED` 반환하도록 검증 추가
* `QuizResultResponse`에 `categoryId`, `difficulty`, `childProfileId` 필드 추가
* 모바일이 sessionId 기반으로 다시 풀기 요청을 구성할 수 있도록 결과 응답 구조 개선
* `categoryName/categoryDisplayName`은 현재 세션 테이블에 저장되지 않아 이번 범위에서는 제외
* Swagger 기반 완료 전/후 결과 조회 및 응답 메타데이터 동작 확인

## 스크린샷/로그

* 완료 후 결과 조회 응답 예시

```json
{
  "sessionId": 13,
  "categoryId": 2,
  "difficulty": "NORMAL",
  "childProfileId": 3,
  "totalQuestionCount": 5,
  "correctCount": 2,
  "totalStar": 7,
  "answers": []
}
```

[P3] 퀴즈 결과 조회 Swagger 에러 코드 문서도 함께 갱신하면 좋겠습니다! :laughing: :laughing: :laughing: 

이번 MR에서 `GET /api/v1/learn/quizzes/sessions/{sessionId}/result`가 완료되지 않은 세션에 대해 `409 LEARN_QUIZ_NOT_COMPLETED`를 반환하도록 개선된 점 확인했습니다. 모바일에서 결과 조회 타이밍을 구분할 수 있어 좋은 변경입니다.

다만 `QuizControllerDocs.getResult()`의 `@ApiErrorCodes`에는 아직 `COMMON_UNAUTHORIZED`, `QUIZ_SESSION_NOT_FOUND`만 포함되어 있어, Swagger 문서만 보면 409 케이스를 알기 어렵습니다.

`LEARN_QUIZ_NOT_COMPLETED`도 문서에 추가해두면 모바일/QA가 결과 조회 실패 케이스를 더 정확히 처리하기 좋을 것 같습니다.

이외 별도로 수정요청할 사항은 없어보입니다! 판단하시어 수정하시거나 곧바로 머지 요청해주셔도 괜찮을거같습니다!
