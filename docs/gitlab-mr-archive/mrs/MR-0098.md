## Description

## 요약

* 학습 API 응답 구조를 모바일 명세에 맞게 수정하고, 로컬 환경에서 Flyway migration이 자동 적용되도록 설정 추가

## 작업 배경

* 관련 이슈: #293
* 모바일 앱에서 `/api/v1/learn/categories` 호출 시 500 오류 발생
* 로컬 DB schema가 최신 migration과 동기화되지 않아 발생한 문제 확인

## 주요 변경 사항

* Flyway migration 자동 적용 설정 추가 (`SPRING_FLYWAY_ENABLED=true`)
* 학습 API 응답 구조를 wrapper 형태로 변경 (`categories`, `levels`, `words`)
* 응답 DTO 필드명을 모바일 명세에 맞게 수정 (`id` → `categoryId`, `wordId`)
* 난이도 enum 값 `MEDIUM` → `NORMAL` 변경 및 migration 추가
