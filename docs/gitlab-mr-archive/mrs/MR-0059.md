## 요약
- 온라인 STT에서 녹음 파일 생성/업로드 흐름을 안정화하고 서버 STT 호출 실패 원인을 분리할 수 있도록 개선


## 작업 배경
- 관련 이슈: #S14P31A404-88


## 주요 변경 사항
- 온라인 STT에서 SpeechRecognizer와 MediaRecorder 동시 사용을 분리하고 MediaRecorder 기반 업로드 흐름으로 변경
- 녹음 start/stop 예외 처리, 빈 파일/무음 파일 전송 방지, VAD 기반 종료 조건 추가
- STT multipart 요청의 locale/audioMimeType을 RequestBody로 명시
- debug 빌드에서 로컬 HTTP 서버 호출을 위해 cleartext 허용
- 스코프 아웃: Clova STT 502 원인 분석 및 백엔드 처리 개선은 후속 작업


## 스크린샷/로그
- Logcat 확인
  - `Cloud STT recording stopped: ..., reason=silence`
  - `Cloud STT recording stopped: ..., reason=no_speech`
  - `Cloud STT recording discarded: speechDetected=false`
  - `Cloud STT upload succeeded: recognized=현 위치, corrected=현 위치, isCorrected=false`
  - `Cloud STT upload failed: API Error: 502 ... code=TRANSLATION_SPEECH_RECOGNITION_FAILED`

- 백엔드 로그 확인
  - `POST /api/v1/translation/speech-to-text`
  - 일부 요청에서 `TRANSLATION_SPEECH_RECOGNITION_FAILED` 발생 확인
