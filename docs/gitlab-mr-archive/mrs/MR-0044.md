## 주요 변경 사항
- **오디오 재생 엔진 구현 (`core/audio`)**: `MediaPlayer`를 활용하여 서버로부터 수신한 음성 데이터를 안정적으로 재생하는 `AudioPlayer` 구축
- **응답 속도(Latency) 최적화**: 
  - 별도의 `audioUrl` 호출 방식 대신, 번역 응답에 포함된 `audioBase64` 데이터를 직접 사용
  - 통신 횟수를 1회(1-RTT)로 단축하여 번역 결과와 음성이 지연 없이 동시에 출력되도록 설계
- **ViewModel 자동 재생 연동**: 번역 성공 시 자동으로 `AudioPlayer`를 호출하는 파이프라인 통합
- **안전한 리소스 관리**: 
  - 재생 시 사용된 임시 캐시 파일 자동 삭제 로직 적용
  - 세션 종료 및 ViewModel 소멸 시 플레이어 리소스 해제(Release) 처리 완료
