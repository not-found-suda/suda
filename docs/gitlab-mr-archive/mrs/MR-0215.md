## Description

## 요약

- 번역 STT를 REST/Streaming 방식으로 선택할 수 있도록 설정 API와 WebSocket streaming gateway를 추가했습니다.

## 작업 배경

관련 이슈: S14P31A404-421\
참고 문서/기획: CLOVA Speech gRPC Streaming 전환 설계

## 주요 변경 사항

- `TRANSLATION_STT_MODE` 환경값과 STT 설정 조회 API 추가
- CLOVA Speech gRPC stream을 열 수 있는 backend gateway 추가
- WebSocket으로 받은 PCM 오디오 chunk를 CLOVA Speech로 전달
- CLOVA transcription 조각을 모바일에 즉시 전달하고 최종 결과는 DB 저장
- 기존 REST STT 흐름은 유지하고 기본값은 `REST`로 설정

## 변경하지 않은 것 / 스코프 아웃

- 모바일 WebSocket 연결 및 녹음 chunk 전송 로직은 후속 작업에서 진행합니다.
- REST STT 제거 또는 기본값 streaming 전환은 성능 비교 후 결정합니다.
- STT 문맥 보정은 이번 작업 범위에 포함하지 않았습니다.

```text
\[P2\] timeout 이후 늦게 도착한 이전 gRPC 콜백이 새 state를 지울 수 있습니다

end timeout task는 `removeStateIfSame(session, state)`로 이전 state인지 확인하고 제거하도록 보완됐습니다. 그런데 timeout으로 이전 state를 제거하고 `state.close()`한 뒤에도, gRPC 쪽에서 이전 stream의 `onCompleted`나 `onError` 콜백이 늦게 도착할 수 있습니다. 이때 `handleClovaCompleted()` / `handleClovaError()`는 현재 WebSocket의 state가 같은지 확인하지 않고 `removeState(session)`를 호출하므로, 같은 WebSocket 연결에서 이미 다음 발화가 `start`된 상태라면 새 발화의 state를 지울 수 있습니다. 완료/에러 콜백에서도 timeout과 동일하게 `removeStateIfSame(session, state)` 방식으로 현재 state가 해당 stream인지 확인한 뒤 정리하는 편이 안전해 보입니다.
```
