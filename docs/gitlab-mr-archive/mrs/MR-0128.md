## Description

## 요약
- 비로그인 사용자도 양방향 소통 기능에 진입할 수 있도록 앱 진입/하단 탭 인증 범위를 조정했습니다.

## 작업 배경
- 관련 이슈: #S14P31A404-224
- 참고 문서/기획:
  - 양방향 소통 기능은 로그인 없이 사용 가능
  - 학습/아이 프로필/리포트/마이페이지 등 개인 데이터 기반 기능은 로그인 필요
  - translation/sign-to-speech, translation/speech-to-text API는 비로그인 허용 정책 확인 완료

## 주요 변경 사항
- AppEntry에서 비로그인 사용자를 Login 대신 Conversation 화면으로 진입하도록 변경
- 하단 탭에서 소통 탭은 비로그인 허용, 보호 기능 탭은 Login으로 유도
- Conversation API가 NoAuth Retrofit을 사용하도록 변경
- 로그아웃 후 Login이 아닌 Conversation으로 이동하도록 백스택 흐름 조정
- 미구현 translation feedback API에 맞춰 신고 버튼을 feature flag로 임시 비활성화

## 스크린샷/로그
- UI 변경: 비로그인 상태에서 앱 실행 시 소통 화면 진입
- API 변경: TranslateApiService가 NoAuth Retrofit 기반으로 호출됨

```text
**\[P2\] 토큰 만료로 세션이 지워져도 하단 탭은 계속 로그인 상태로 판단할 수 있습니다**

```
