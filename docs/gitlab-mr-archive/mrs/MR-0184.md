## Description

## 요약

- 로그인된 사용자의 회원 탈퇴 API를 추가하고, 탈퇴 시 토큰 무효화 정책을 반영했습니다.

## 작업 배경

관련 이슈: S14P31A404-357\
참고 문서/기획: 유저 도메인 2순위 API 요청, 회원 탈퇴 정책

## 주요 변경 사항

- `DELETE /api/v1/users/me` 회원 탈퇴 API 추가
- 비밀번호 로그인 계정은 현재 비밀번호 검증 후 탈퇴 처리
- 소셜 전용 계정 구분을 위한 `passwordLoginEnabled` 필드 추가
- 탈퇴 시 사용자 `active=false` 처리 및 refresh/access token 무효화
- 회원 탈퇴 정책 및 토큰 무효화 테스트 추가

## 스크린샷/로그

- Swagger에서 회원 탈퇴 API `204 No Content` 응답 확인
- 탈퇴 후 동일 계정 로그인 시 `403 AUTH_INACTIVE_ACCOUNT` 응답 확인

## 메모

- 회원 탈퇴는 hard delete가 아니라 `active=false` soft delete 방식으로 처리했습니다.
- 비밀번호 로그인 계정은 탈퇴 요청 시 비밀번호가 필요하고, 소셜 전용 계정은 access token 인증을 기준으로 탈퇴 처리합니다.
- 탈퇴 성공 후 해당 유저의 refresh token을 Redis에서 삭제하고, 기존 access token도 invalid-before 기준으로 차단되도록 처리했습니다.
