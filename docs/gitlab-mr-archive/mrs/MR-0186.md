## Description

## 요약

- 로그인하지 못하는 사용자를 위한 비밀번호 재설정 요청/검증/확정 API를 추가했습니다.

## 작업 배경

관련 이슈: S14P31A404-354\
참고 문서/기획: 모바일 유저 도메인 2순위 요청, 비밀번호 재설정 플로우

## 주요 변경 사항

- `POST /api/v1/auth/password-reset/request` 인증 코드 요청 API 추가
- `POST /api/v1/auth/password-reset/verify` 인증 코드 검증 및 resetToken 발급 API 추가
- `POST /api/v1/auth/password-reset/confirm` 새 비밀번호 재설정 확정 API 추가
- Redis 기반 인증 코드/resetToken 저장 구조 추가
- 비밀번호 재설정 성공 시 기존 사용자 토큰 무효화 처리

## 스크린샷/로그

- UI 변경 없음
- API 변경으로 Swagger 또는 응답 캡처 첨부 예정

## 메모

- 현재 메일 발송 인프라가 없어 인증 코드는 `PasswordResetCodeSender` 인터페이스를 통해 로그 출력 방식으로 임시 처리했습니다.
- 추후 SMTP/메일 API 연동 시 sender 구현체만 교체하면 비밀번호 재설정 흐름은 유지할 수 있습니다.
- 가입 여부 노출을 막기 위해 재설정 요청 API는 존재하지 않는 이메일이어도 `204 No Content`를 반환합니다.
- 소셜 전용 계정과 비활성 계정에는 재설정 인증 코드를 발급하지 않습니다.
