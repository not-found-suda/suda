전체적으로 의도한 방향대로 잘 정리된 것 같습니다. :thumbsup_tone4: :thumbsup_tone4: 

TranslationServiceImpl에서 Gemini 보정 호출이 제거되고, STT 응답 DTO 구조는 유지하면서 correctedText=recognizedText, corrected=false로 내려가도록 되어 있어 모바일 호환성도 문제 없어 보입니다. 모바일 쪽은 기존 correctedText 사용 흐름을 유지해도 동작할 것 같습니다. :grin: 

문서/정리 관점에서 두 가지만 확인 부탁드립니다.

1. TranslationApiDocs의 STT 설명이 아직 "변환 및 보정", "보정된 문장"으로 남아 있어 현재 동작과 조금 다르게 읽힙니다. 보정 단계가 제거됐으니 STT 결과를 그대로 반환한다는 설명으로 맞추면 좋겠습니다.

2. TRANSLATION_CHILD_SPEECH_CORRECTION_FAILED 에러 코드는 이제 발생 경로가 사라진 것으로 보여 정리 여지가 있어 보입니다. 향후 재사용 예정이면 유지해도 괜찮고, 아니라면 이번 제거 범위에 같이 포함하면 더 깔끔할 것 같습니다.

기능 동작 자체는 문제 없어 보여서 위 내용은 문서/정리 성격의 코멘트입니다. 확인 부탁드립니다 :smile:
