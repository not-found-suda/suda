**[P4] 이미지 변환 프로세스 최적화 및 UI 부하 제어 제안**

홍헌님, 파이프라인의 성능을 실시간으로 튜닝하고 모니터링할 수 있는 디버그 시스템이 정말 좋다고 생각합니다 :grin: :grin: :grin: 

특히 런타임에 파라미터를 조절할 수 있는 TuningCard는 우리 팀의 개발 생산성을 크게 높여줄 것 같아요.


실제 배포 및 장시간 테스트를 고려하여 몇 가지 성능 최적화 제안을 드립니다:

이미지 변환 오버헤드 최적화 (Critical): toUprightBitmap()에서 JPEG 압축/해제를 거쳐 비트맵을 생성하는 방식은 CPU와 메모리 점유율이 매우 높습니다. MediaPipe가 지원하는 ImageProxy 직접 입력 방식을 활용하거나, 비트맵이 꼭 필요하다면 JNI 기반의 libyuv 또는 RenderScript 등을 활용하여 변환 과정을 경량화하면 발열 문제를 크게 개선할 수 있을 것 같습니다.

UI 리컴포지션 스로틀링: 현재 모든 지표가 매 프레임 갱신되고 있는데, SignDebugViewModel에서 throttleFirst 등을 사용하여 UI 업데이트 주기를 초당 5~10회 정도로 조절하면 디버깅 모드에서의 전체적인 기기 부하를 낮출 수 있습니다.

제가 해당 작업에서 홍헌님만큼 이해도가 높지 않기 때문에, 이 제안이 유효한지는 홍헌님이 판단해서 진행해주시면 좋을 것 같습니다.
