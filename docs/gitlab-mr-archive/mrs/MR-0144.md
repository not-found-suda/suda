### nej0720 at 2026-05-12T15:54:29.546+09:00 (comment)

```text
\[P3\] 백엔드 count 타입과 모바일 DTO 타입을 맞추는 게 안전해 보입니다

백엔드 리포트 응답에서는 `completedSessionCount`, `totalQuestionCount`, `totalCorrectCount`, `attemptCount`, `wrongCount`가 `Long` 타입으로 내려오는데, 모바일 DTO에서는 해당 필드들을 `Int`로 받고 있습니다.

현재 값이 크지 않다면 바로 문제가 되지는 않겠지만, API 계약 기준으로는 타입이 다르기 때문에 추후 누적 데이터가 많아지거나 Gson 파싱/모델 변경 시 예기치 않은 문제가 생길 수 있습니다. 백엔드 응답 타입에 맞춰 모바일 DTO와 domain model도 `Long`으로 맞추는 편이 더 안전해 보입니다.
```
