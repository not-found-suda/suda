## 요약
- 학습 카테고리/단어 카드에서 이미지 URL을 로딩하고 실패 시 대체 UI를 표시하도록 구현했습니다.

## 작업 배경

## 주요 변경 사항

- 카테고리별 단어 카드에 `imageUrl` 이미지 로딩을 적용했습니다.
- 스코프 아웃: QuizQuestionRoute는 팀원 작업 충돌 방지를 위해 이번 티켓에서 제외했습니다.

### hh001204 at 2026-05-07T12:14:34.598+09:00 (comment)

```text
**P4: 이미지 fallback의 접근성 문구를 조금 더 구분해도 좋겠습니다**

`AppNetworkImage`는 이미지가 없거나 실패했을 때 첫 글자 fallback을 보여주고 있습니다. 화면상으로는 자연스럽지만, 실제 이미지와 fallback이 같은 `contentDescription`을 공유하고 있어서 접근성 도구에서는 이미지가 로드된 것처럼 읽힐 수 있습니다.

큰 문제는 아니지만, 나중에 접근성까지 다듬는다면 이미지 실패/대체 UI일 때는 `contentDescription`을 null로 두거나 별도 문구를 주는 방식도 고려해볼 수 있을 것 같습니다.
```
