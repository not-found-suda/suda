요청드렸던 current question 응답의 targetText 추가가 잘 반영된 것 확인했습니다. :smile: 

QuizCurrentQuestionResponse에 targetText가 추가되었고, QuizService에서 word.displayText를 targetText로 내려주도록 되어 있어 답변 제출/결과 API의 targetText 계약과도 일관성이 맞아 보입니다. displayText도 Learn 엔티티에서 NOT NULL이라 모바일 표시/채점 기준으로 사용하기에 안전해 보입니다.

이 MR이 머지되면 모바일 쪽에서는 기존 word 목록 선조회 의존을 줄이고 current question 응답의 targetText를 우선 사용하는 방향으로 후속 정리하겠습니다. 고생 많으셨습니다!
