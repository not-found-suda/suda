## Description

## 요약

- 리포트 Summary API가 날짜 범위 조회 시 500을 반환하던 JPQL 조립 오류를 수정

## 작업 배경

관련 이슈: S14P31A404-371\
참고 문서/기획: 모바일 리포트 조회/필터 안정화 요청

## 주요 변경 사항

- Summary 조회 중 호출되는 `findWeakWords(...)` JPQL에서 `WHERE` 절과 `GROUP BY`/`AND` 절이 붙는 문제 수정
- 동일한 동적 JPQL 조립 패턴이 있는 카테고리 조회 쿼리도 함께 보정
- 완료 기록 없음, 취약 단어 없음, 취약 단어 있음 케이스에 대한 Repository 통합 테스트 추가

변경하지 않은 것 / 스코프 아웃:

- 모바일 요청 형식 및 API 명세는 변경하지 않음
- Summary 응답 DTO 구조는 변경하지 않음

## 스크린샷/로그

API 오류 원인 재현 로그:

```text
org.hibernate.query.SyntaxException:
mismatched input 'BY'
...
AND s.status = :completedGROUP BY w.id, w.word, ...
```

## **테스트 결과**

* Repository 통합 테스트 추가 및 통과
  * 기간 내 완료 기록 없음
  * 완료 세션은 있지만 취약 단어 없음
  * 취약 단어 row가 실제로 있는 경우
* 로컬 전체 테스트 통과

## 메모

- 500 발생 지점은 summarize(...)나 findLatestCategory(...)가 아니라 Summary 응답 생성 중 항상 호출되는 findWeakWords(...) JPQL 파싱 단계였습니다.
- 동일한 문자열 조립 방식의 findCategories(...)도 예방 차원에서 함께 수정했습니다.
