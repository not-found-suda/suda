## Description

## 요약

* 네이버 OAuth 로그인을 Android SDK provider access token 방식으로 전환했습니다.

## 작업 배경

- 관련 이슈: S14P31A404-340
- 참고 문서/기획:

## 주요 변경 사항

* POST /api/v1/auth/oauth/naver 요청 Body를 providerAccessToken 기반으로 변경했습니다.
* 백엔드의 네이버 authorization code/token exchange 로직을 제거했습니다.
* 전달받은 provider access token으로 네이버 프로필 API를 호출하도록 수정했습니다.
* OAuth 오류 코드를 OAUTH_INVALID_PROVIDER_TOKEN 기준으로 정리했습니다.
* Swagger 설명을 네이버 Android SDK 토큰 방식에 맞게 수정했습니다.

변경하지 않은 것 / 스코프 아웃

* 네이버 Android SDK 로그인 및 access token 획득은 모바일 파트 작업입니다.
* 소셜 계정 생성/매핑 및 우리 서비스 JWT 발급 흐름은 기존 구조를 유지했습니다.

## 스크린샷/로그

* API는 네이버 Android SDK에서 발급받은 실제 provider access token이 있어야 성공 응답 캡처가 가능합니다.\
  현재 백엔드 단독으로는 컴파일/테스트 통과까지 확인했습니다.
