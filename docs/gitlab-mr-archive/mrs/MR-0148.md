### nej0720 at 2026-05-12T19:51:14.040+09:00 (comment)

```text
\[P3\] 퀴즈 기록 화면에서 아이 선택 후 홈으로 빠질 수 있습니다

`ReportQuizSessionsRoute`의 아이 선택 버튼은 `ChildSelect`로 이동하지만, 아이 선택 완료 후 복귀 로직은 현재 `previousRoute`가 `ReportHome`일 때만 리포트 화면으로 돌아가도록 되어 있습니다.

그래서 퀴즈 기록 화면에서 `Missing`/`NotFound` 상태로 `아이 선택하기`를 누른 뒤 프로필을 선택하면, `previousRoute`가 `ReportQuizSessions`라서 현재 화면으로 돌아오지 않고 `Home`으로 fallback될 수 있을 것 같습니다.

퀴즈 기록 화면에서도 아이 선택 후 원래 리포트 화면으로 돌아오도록 `ReportQuizSessions` route도 함께 처리하면 더 자연스러울 것 같습니다!
```

```text
\[P3\] 더 보기 재시도 성공 후 이전 에러 메시지가 다시 표시될 수 있습니다

`loadMoreQuizSessions()`는 요청 시작 시 `loadMoreErrorMessage`를 `null`로 지우지만, 성공 처리에서는 요청 시작 전에 캡처한 `quizSessionsState`를 다시 `copy`하고 있습니다.

그래서 이전에 더 보기 요청이 실패해서 에러 메시지가 있던 상태에서 재시도에 성공하면, 성공 후에도 이전 에러 메시지가 다시 표시될 수 있을 것 같습니다.

성공 케이스에서도 `loadMoreErrorMessage = null`을 명시하거나, 현재 `uiState`의 `Success` 상태를 기준으로 `copy`하면 더 보기 성공 후 에러 문구가 남지 않을 것 같습니다!
```
