## Description

## 요약
- 퀴즈 답변 제출 흐름을 기존 JSON 텍스트 제출에서 서버 채점용 multipart 음성 파일 제출 방식으로 전환했습니다.

## 작업 배경
- 관련 이슈: #S14P31A404-303
- 참고 문서/기획:
  - 백엔드 퀴즈 답변 제출 API 계약
  - `POST /api/v1/learn/quizzes/sessions/{sessionId}/answers`
  - `Content-Type: multipart/form-data`
  - `questionId` form field
  - `audioFile` file part

## 주요 변경 사항
- 퀴즈 답변 제출 API를 `multipart/form-data` 기반으로 변경했습니다.
- 퀴즈 녹음 결과를 WAV 파일로 생성하고 서버에 `audioFile` part로 전송하도록 연결했습니다.
- 서버 채점 응답의 `recognizedText`, `isCorrect`, `star`, `feedback`을 퀴즈 UI에 반영하도록 정렬했습니다.
- 기존 로컬 STT/로컬 채점 기반 제출 흐름을 제거하고 서버 응답을 기준으로 상태를 갱신하도록 변경했습니다.
- 변경하지 않은 것 / 스코프 아웃: multipart 음성 파일 오프라인 큐 재전송은 후속 티켓으로 분리합니다.

## 스크린샷/로그
- API 연동 변경 작업으로 별도 UI 스크린샷은 생략합니다.

확인 감사합니다! 현재 `QUIZ_AUDIO_FILE_NAME`은 `AndroidAudioRecorder.start()`에 넘기는 base name이고, 실제 파일은 `AndroidAudioRecorder` 내부에서 `"$fileName.wav"`로 생성되어 `quiz_answer_audio.wav`가 됩니다.

녹음 구현도 `AudioRecord`로 PCM 데이터를 쓰고 WAV header를 생성/마무리하는 구조라 실제 포맷과 `audio/wav` MIME은 일치하는 것으로 확인했습니다. multipart filename도 `file.name`을 사용해서 서버에는 `.wav` 확장자가 포함된 파일명으로 전달됩니다.

다만 코드만 보면 base name인지 헷갈릴 수 있어서, 필요하면 상수명을 `QUIZ_AUDIO_BASE_FILE_NAME`처럼 바꾸거나 주석을 추가해서 의도를 더 명확히 하겠습니다. :grin:
