## 요약

`STT 요청에서 미지원 오디오 포맷을 차단하고 Clova 호출 로그를 보강`

## 작업 배경

* 관련 이슈: #14P31A404-80
* 참고 문서/기획: Clova CSR STT 지원 오디오 포맷 확인

## 주요 변경 사항

* STT 요청 허용 MIME 타입을 WAV 계열로 제한
* `audio/mp4`, `.m4a` 요청은 `TRANSLATION_INVALID_AUDIO`로 거절
* STT 요청 수신 로그 추가
* Clova STT 호출 전후 로그 추가
* Secret/API Key/오디오 바이너리는 로그 제외

## 스크린샷/로그

* `.m4a` 업로드 시 400 응답 확인

```json
{
  "status": 400,
  "title": "유효하지 않은 음성 파일",
  "code": "TRANSLATION_INVALID_AUDIO"
}
```
