### soshyzx123 at 2026-05-13T14:01:04.449+09:00 (comment)

```text
[P1] 대화 화면에서 번역 메시지 자막이 더 이상 렌더링되지 않습니다. :grin: 

`ConversationRoute.kt`에서 일반 화면의 `SignRecognitionArea` 호출은 `messages = uiState.messages`를 넘기지만 `showSubtitles = false`로 고정되어 있습니다. 그런데 `SubtitleList`는 `SignRecognitionArea` 내부의 `if (showSubtitles)` 블록에서만 렌더링되고, 하단 영역에는 `ParallelInputStatusCard`와 `GlossRecognitionArea`만 표시됩니다.

이 상태면 수어/음성 번역 결과와 기존 대화 메시지를 사용자가 볼 수 없고, 메시지 피드백 진입도 사실상 불가능해집니다. 카메라 비율을 줄인 새 레이아웃을 유지하더라도 `SubtitleList(messages = uiState.messages, ...)`를 하단 영역 등에 다시 배치해야 할 것 같습니다.
```

### soshyzx123 at 2026-05-13T14:01:13.522+09:00 (comment)

```text
[P3] 전체화면 종료 시 이전 orientation 값을 복원하는 편이 안전합니다. :smiley: :smiley: 

`ForceLandscapeOrientationEffect()`가 진입 시 landscape로 강제한 뒤, dispose에서 항상 `SCREEN_ORIENTATION_PORTRAIT`로 되돌리고 있습니다. 기존 Activity가 `UNSPECIFIED`, `SENSOR`, manifest 설정 등 다른 방향 정책을 사용 중이었다면 전체화면을 닫은 뒤 앱 방향 정책이 portrait로 고정될 수 있습니다.

진입 전에 `activity.requestedOrientation` 값을 저장해두고 dispose에서 그 값을 복원하면 다른 화면의 방향 정책에 영향을 덜 줄 수 있을 것 같습니다.
```
