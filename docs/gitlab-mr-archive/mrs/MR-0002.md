```text
p3.
테스트 환경까지 미리 꼼꼼하게 챙겨주셔서 감사합니다! 👍👍
다만, 의존성 설정 중 추후 빌드 단계에서 에러를 유발할 수 있는 부분이 있어 수정을 요청드립니다.

현재 로컬에서 서버를 띄우셨을 때는 인텔리제이의 실행 버튼을 쓰셨거나 테스트를 스킵하셨다면 정상 작동했을 수 있습니다.
하지만 나중에 테스트 코드를 작성하고 터미널에서 순수하게 ./gradlew test나 ./gradlew build 명령어를 돌리게 되면 에러가 발생하게 됩니다.

이유: 스프링 부트에서는 validation이나 webmvc 전용 테스트 라이브러리를 별도로 쪼개서 제공하지 않습니다.
(현재 적어주신 이름의 라이브러리는 Maven 중앙 저장소에 존재하지 않아 Gradle이 다운로드할 수 없습니다.)
대신 JUnit, Mockito, AssertJ 등 모든 핵심 테스트 도구들을 spring-boot-starter-test라는 단일 패키지 하나에 묶어서 제공하고 있습니다.

아래와 같이 표준 스타터 하나로 통합해 주시면 추후 파이프라인에서도 안전하게 빌드가 통과될 것 같습니다.
`` `
// 수정 전 (추후 에러 발생)
// testImplementation 'org.springframework.boot:spring-boot-starter-validation-test'
// testImplementation 'org.springframework.boot:spring-boot-starter-webmvc-test'

// 수정 후 (표준 단일 패키지로 통합)
testImplementation 'org.springframework.boot:spring-boot-starter-test'
`` `
```
