```text
\[P1\] 전역 ObjectMapper 교체로 LocalDate/LocalDateTime 응답 직렬화가 깨질 수 있습니다

이번 브랜치에서 new ObjectMapper()를 반환하는 전역 ObjectMapper 빈이 추가됐는데, 이렇게 하면 Spring Boot가 기본으로 구성해주는 Jackson 설정이 밀려날 수 있습니다. 현재 백엔드 DTO에는 LocalDate, LocalDateTime 응답 필드가 여러 곳에 있어서, JavaTimeModule이 등록되지 않은 plain ObjectMapper가 전역으로 사용되면 해당 API들이 런타임에 직렬화 실패할 수 있습니다. WebSocket handler에서 ObjectMapper 주입이 필요했던 거라면 Spring Boot 기본 ObjectMapper를 그대로 주입받거나, 직접 빈을 만들더라도 Jackson2ObjectMapperBuilder/JavaTimeModule 등 기존 설정을 유지하는 방식이 안전해 보입니다.

\[P2\] WebSocket start 완료 전에 오디오 청크가 먼저 전송될 수 있습니다

streamingSttClient.start() 직후 바로 startPcmStream을 시작해서 오디오 청크를 보내고 있는데, 실제 start 메시지는 WebSocket onOpen 콜백에서 비동기로 전송됩니다. OkHttp WebSocket은 연결 완료 전에도 send를 큐에 넣을 수 있어서, 타이밍에 따라 binary audio가 start 메시지보다 먼저 서버에 도착할 수 있고, 서버는 아직 STT state가 없어 Streaming STT session is not started. 에러를 반환할 수 있습니다. Started 또는 Configured 이벤트를 받은 뒤 녹음을 시작하거나, 그 전까지 오디오 청크를 버퍼링/차단하는 흐름으로 맞추면 streaming STT가 더 안정적으로 동작할 것 같습니다.
```
