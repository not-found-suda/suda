```text
p3. 모델 구조 깔끔하고 좋습니다! 👍👍

PyTorch의 BiLSTM에서 out[:, -1, :]를 가져오는 부분에 함정이 하나 있습니다잇!
out 텐서에서 Forward 방향은 인덱스 -1이 전체 시퀀스를 다 본 '마지막 상태'가 맞지만,
Backward 방향은 역순으로 처리되기 때문에 인덱스 0이 전체를 다 본 '마지막 상태'가 됩니다.
즉, 현재 코드는 Backward 기준으로는 단 1개의 프레임만 본 초기 상태를 가져와서 분류기에 넣고 있는 셈이에요.

이를 해결하기 위해 out 대신 hidden state(h_n)를 직접 사용해서 Concat 하거나, 아래 코드와 같 수정해 보는 것을 제안드림다.
`` ` python 
def forward(self, x: torch.Tensor) -> torch.Tensor:
    out, (h_n, c_n) = self.lstm(x)

    # h_n shape: (num_layers * 2, batch, hidden_size)
    # 가장 마지막 레이어의 forward와 backward 상태를 가져와서 합침
    forward_hidden = h_n[-2, :, :]
    backward_hidden = h_n[-1, :, :]
    last = torch.cat((forward_hidden, backward_hidden), dim=1)

    last = self.dropout(last)
    return self.fc(last)
`` `
```
