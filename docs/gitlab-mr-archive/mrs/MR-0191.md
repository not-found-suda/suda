**P3: AI 서버 포트는 기본 compose에서 외부 공개하지 않는 편이 안전해 보입니다**

현재 ai-server가 ports로 host 8000번 포트에 노출되어 있습니다. 백엔드가 같은 Docker network 안에서 `http://ai-server:8000`으로 호출하는 구조라면 외부 포트 공개는 필수는 아닙니다.

compose를 하나로 유지할 계획이라면 기본값은 ports 대신 expose로 두고, AI 서버를 직접 테스트해야 할 때만 dev override compose로 ports를 여는 방식이 더 안전해 보입니다.

개발용이라면 괜찮지만, 배포 compose에서도 쓸 수 있다면 expose만 쓰거나 `127.0.0.1:${AI_SERVER_PORT}:8000`처럼 접근 범위를 줄이는 편이 안전합니다.
```

**P2: 공개 추론 API에 입력 크기 상한이 부족합니다**

`/api/v1/sign/**`가 `permitAll`이고, `sequenceLength`, `featureDimension`, `features` 크기에 상한이 거의 없습니다. 큰 배열을 보내면 메모리/AI 서버 부하가 커질 수 있고, `sequenceLength * featureDimension`도 `int overflow` 가능성이 있습니다.
기본 규격이 `30 * 332`라면 @Max, 고정 규격 검증, 요청 `body size` 제한 중 하나는 두는 게 좋겠습니다.
```
