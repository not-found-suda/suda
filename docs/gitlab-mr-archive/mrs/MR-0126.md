## Description

## 요약
- 보호자가 리포트 기능에 접근할 수 있도록 하단 탭과 리포트 홈 레이아웃을 추가했습니다.

## 작업 배경
- 관련 이슈: #S14P31A404-242
- 참고 문서/기획:
  - 리포트 API는 아직 확정 전이며, 현재 백엔드 구상 방향은 summary / weak-words / category-progress 기준입니다.

## 주요 변경 사항
- 하단 네비게이션에 `리포트` 탭을 추가했습니다.
- `ReportHome`, `ReportSummary`, `ReportWeakWords`, `ReportCategoryProgress` 라우트를 추가했습니다.
- 활성 아이 정보를 표시하는 보호자 리포트 홈 화면을 구현했습니다.
- 학습 요약 / 자주 틀리는 단어 / 카테고리별 학습 진행도 카드를 placeholder 상세 화면과 연결했습니다.
- 실제 리포트 API 연동, Repository/DTO 생성, 차트/통계 계산은 스코프 아웃했습니다.

## 스크린샷/로그
- UI 변경 사항으로 리포트 탭 및 리포트 홈/placeholder 화면 확인 필요

```text
**P2: 선택된 아이 영역의 버튼 레이아웃이 깨질 수 있습니다**

`ReportActiveChildSection`에서 선택된 아이 정보를 Row로 배치하고 있는데, 오른쪽의 `AppSecondaryButton`은 내부에서 `fillMaxWidth()`를 강제하고 있습니다. 이 때문에 ‘아이 변경’ 버튼이 Row의 남은 공간을 크게 차지하면서 아이 이름 영역을 밀어낼 수 있습니다.

이 화면에서는 작은 보조 버튼처럼 보여야 하므로, 여기서는 `Material3 OutlinedButton/TextButton`을 직접 쓰거나 공통 버튼이 `full-width`를 강제하지 않는 형태로 분리하면 더 안정적일 것 같습니다.
```
