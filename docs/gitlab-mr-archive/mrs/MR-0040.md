## Description

## **요약**

* translation 도메인에 자녀 발화용 speech-to-text API를 추가하고, 원문 STT 텍스트와 보정 텍스트를 함께 반환하는 mock 흐름 및 입력 검증/예외 처리를 구현했습니다.

## **작업 배경**

* 관련 이슈: #S14P31A404-63
* 참고 문서/기획:
  * mvp-spec.md
  * features.md

## **주요 변경 사항**

* speech-to-text 엔드포인트 및 SpeechToTextResponseDto 추가
* TranslationController, TranslationService, TranslationServiceImpl에 자녀 발화 흐름 추가
* TranslationApiDocs에 자녀 API Swagger 문서 추가
* MIME type 검증 및 audio/mp4 지원 추가
* MissingServletRequestPartException, locale 검증 등 입력 예외 처리 보완

## **변경하지 않은 것 / 스코프 아웃**

* Clova STT / Gemini 실제 외부 연동은 아직 미구현
* 현재 speech-to-text는 mock 응답 기반
* 파일 내용 기반 고급 검증 로직은 이번 MR 범위 제외

## **스크린샷/로그**

* ./gradlew spotlessApply --no-daemon 적용
* ./gradlew spotlessCheck --no-daemon 통과
* ./gradlew build -x test --no-daemon 통과

## **테스트 결과**

* 단위 테스트는 아직 작성하지 않음
* 로컬 포맷 검사 및 테스트 제외 빌드 통과

## **작성자 체크리스트**

*  Secret/Env/개인정보가 포함되지 않음
  * [x]  불필요한 로그/디버그 코드 제거
  * [x]  로컬 빌드/런 정상
  * [x]  관련 문서/주석/README 업데이트(필요 시)
  * [x]  이슈/Jira 상태 업데이트(In Review/Code Review)

## **메모**

* 현재 응답은 recognizedText, correctedText를 함께 반환하도록 구성했습니다.
* locale은 현재 ko-KR만 허용하며, 잘못된 값은 TRANSLATION_INVALID_LOCALE로 처리합니다.
* 실제 Clova STT / Gemini 연동은 후속 작업에서 TranslationServiceImpl 내부에 연결 예정입니다.
