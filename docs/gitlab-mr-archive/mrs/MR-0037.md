**P2: 번역 요청이 중복 실행되는 흐름, 한번 확인해보시면 어떨까요?**

현재 인식 단어가 3개 이상 쌓인 뒤 새 Prediction이 들어올 때마다 번역 요청이 다시 실행될 수 있는 구조인 것 같아요!

예를 들어 [엄마, 밥, 먹다]로 요청 A를 보낸 다음, A 응답이 오기 전에 [엄마, 밥, 먹다, 좋아]로 요청 B가 또 나가는 상황이 생길 수 있어요. 이때 B가 먼저 성공하고 A가 나중에 도착하면, 더 오래된 A의 응답이 최신 번역 결과를 덮어버릴 가능성이 있답니다

실시간으로 계속 번역을 갱신하려는 의도라면 지금 방향도 충분히 이해돼요! 다만 한 문장 단위 번역을 기대하는 흐름이라면, 아래 두 가지 방법 중 하나를 고려해보시면 더 안전할 것 같아요.

- 번역 요청 중에는 추가 요청을 막는 isTranslating 같은 상태값을 두는 방법
- Utterance 단위로 요청을 보내는 방식으로 변경하는 방법

혹시 의도하신 흐름이 어느 쪽인지 확인해주시면 좋겠어요~ 😄🙌
