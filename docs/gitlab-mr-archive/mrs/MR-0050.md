## Description

## 요약
- 농인(부모)과 청인(자녀)의 대화 내역을 실시간으로 시각화하는 자막 UI 구현

## 작업 배경
- 관련 이슈: #S14P31A404-146
- 수어 인식 결과와 음성 인식 결과를 한 화면에서 보여주어 원활한 양방향 소통을 지원하기 위함

## 주요 변경 사항
- **대화형 자막 UI 시스템 구축**:
  - `ChatMessage` 도메인 모델을 도입하여 발화 주체와 메시지 상태(교정 중/완료) 관리
  - 농인(왼쪽/회색)과 청인(오른쪽/브랜드 컬러)을 시각적으로 구분하는 커스텀 말풍선 구현
- **실시간 데이터 연동 및 최적화**:
  - `LazyColumn`을 활용하여 대화 내역이 많아져도 안정적인 렌더링 보장
  - 새로운 메시지 유입 시 자동으로 최하단으로 이동하는 **Auto-scroll** 기능 적용
  - AI 교정 중(Pending) 상태를 반투명 애니메이션 효과로 처리하여 사용자 피드백 강화
- **화면 통합 레이아웃**:
  - `SignRecognitionScreen`(카메라 레이어) 상단에 자막 리스트를 오버레이하여 전문적인 영상 통화/번역 UI 구축

```text
P2: TTS를 재생하기 전에 STT를 잠시 멈추면 좋겠습니다

오디오 재생 로직을 `playBase64()`로 분리한 점은 좋아 보입니다.

다만 지금은 앱이 TTS로 소리를 낼 때도 STT가 계속 듣고 있을 수 있습니다. 그러면 앱이 스피커로 낸 소리를 STT가 다시 듣고, 사용자가 말한 것처럼 받아쓸 가능성이 있습니다.

`playBase64()`를 호출하기 전에 `sttEngine.stopListening()`을 한 번 호출하면 이 흐름을 더 안전하게 막을 수 있을 것 같습니다.
```
