## Description

## 요약
- 보호자가 이메일/비밀번호로 로그인하고, 세션 저장 후 앱 진입 화면으로 이동할 수 있도록 구현

## 작업 배경
- 관련 이슈: #S14P31A404-232
- 참고 문서/기획:
  - 백엔드 로그인 API: POST /api/v1/auth/login
  - 현재 백엔드는 refreshToken을 Set-Cookie로 내려주므로, 이번 티켓에서는 accessToken 저장까지만 처리

## 주요 변경 사항
- 로그인 화면 UI, 입력 검증, 로딩/오류 상태를 구현
- NoAuth Retrofit 기반 로그인 API 계층과 DI를 추가
- 로그인 성공 시 기존 토큰을 정리하고 accessToken을 저장한 뒤 활성 아이 프로필 여부에 따라 라우팅
- Login → Home/ChildSelect 이동 시 Login 화면을 백스택에서 제거
- 스코프 아웃: 회원가입 화면, 네이버 SDK 연동, refreshToken body 연동, 백엔드 수정

## 스크린샷/로그
- UI 변경: 로그인 화면 추가

![image](/uploads/8b7aaa9d2964a87e8e23dba7d868033e/image.png){width=246 height=600}
- API 연동: POST /api/v1/auth/login
- refreshToken은 현재 Cookie 방식이라 모바일 저장 연동은 후속 백엔드 협의 후 진행

## 메모
- 현재 백엔드 로그인 응답은 accessToken, tokenType, expiresIn만 body로 내려오고, refreshToken은 Set-Cookie 방식임
- 모바일 refresh 연동을 위해 백엔드에 refreshToken body 응답 또는 모바일 전용 auth endpoint 협의 필요
- 회원가입 이동과 네이버 로그인은 후속 티켓에서 연결 예정
