## 요약

아이별 리포트 요약과 취약 단어 목록을 조회할 수 있도록 API를 구현했습니다.

## 주요 변경 사항

- `GET /api/v1/children/{childId}/reports/summary` 리포트 요약 조회 API 추가
- `GET /api/v1/children/{childId}/reports/weak-words` 취약 단어 목록 조회 API 추가
- 퀴즈 세션/답변 기준으로 정답률, 평균 별점, 최근 퀴즈, 취약 단어 집계 구현
- summary 응답에 취약 단어 TOP 5 포함
- weak-words 목록은 page/size 기반 페이지 응답으로 제공

## 변경하지 않은 것 / 스코프 아웃

- 카테고리별 리포트 요약 API는 별도 이슈에서 구현 예정입니다.
- 별도 집계 테이블이나 집계 컬럼은 추가하지 않았습니다.
- 학습 카드 화면 기록은 현재 저장 데이터가 없어 집계 대상에 포함하지 않았습니다.

[P2] `ApiErrorCodeCatalog`에 신규 리포트 에러 코드가 등록되지 않았습니다. :smiley: 

`ReportApiDocs`에서는 `REPORT_NOT_FOUND`를 선언하고 있는데, `ApiErrorCodeCatalog`에는 `ReportErrorCode`가 등록되어 있지 않아 OpenAPI 커스터마이저가 unknown code로 처리할 것 같습니다. 이 경우 404 에러 예시/응답 문서가 누락됩니다.

[P2] 취약 단어 조회가 전체 결과를 메모리에 올린 뒤 페이지네이션하고 있습니다. :grin: 

`ReportQuizSessionQueryRepository.findWeakWords()`에서 `query.getResultList()`로 전체 그룹 결과를 조회한 다음 `subList`로 페이지를 자르고 있습니다. `/weak-words` 목록 조회뿐 아니라 요약 API의 TOP 5 조회에서도 전체 취약 단어 결과를 모두 materialize하게 되어, 답변 데이터가 쌓이면 응답 지연이나 메모리 사용이 커질 수 있습니다.

데이터 조회 쿼리는 `setFirstResult` / `setMaxResults`로 DB 페이지네이션을 적용하고, `totalElements`는 별도 count 쿼리로 계산하는 방식으로 보완하면 좋겠습니다. :thumbsup_tone4:
