```text
215번 추가 변경분 확인했습니다. :grin: 

답변 저장 실패를 기존 Error로 끝내지 않고 SaveFailed 상태로 분리해서, 사용자가 같은 답변 저장을 다시 시도할 수 있게 만든 방향이 좋아 보입니다. 일시적인 네트워크 오류나 서버 순간 실패에는 UX가 더 자연스러울 것 같습니다. :grinning: 


다만 현재는 submitAnswer 실패가 전부 SaveFailed로 매핑되고, SaveFailed 상태에서는 녹음 버튼이 비활성화된 채 “답변 저장 다시 시도”만 가능해 보입니다. 네트워크 오류처럼 재시도 가능한 실패라면 괜찮지만, 400/401/404/409처럼 같은 요청을 반복해도 복구되지 않는 실패까지 이 상태가 되면 사용자가 같은 실패 요청만 반복하는 흐름에 갇힐 수 있을 것 같습니다.

가능하면 retry 가능한 오류만 SaveFailed로 두거나, SaveFailed 상태에서도 다시 말하기/세션 재시작 같은 탈출 경로를 하나 더 제공하면 더 안전해 보입니다.

그 외 215번에서 추가한 상태 분리와 버튼 문구/상태 전환 자체는 깔끔해 보입니다. 214번에서 이야기했던 백엔드 API 계약 관련 이슈는 상위 브랜치 이슈로 별도 관리하면 될 것 같습니다.

전체적으로 좋은 변경인거같습니다. :grin: :thumbsup_tone4:
```
