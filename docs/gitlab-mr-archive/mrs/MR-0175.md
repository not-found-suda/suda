## Description

## 요약

- 로그인된 사용자가 현재 비밀번호 확인 후 새 비밀번호로 변경할 수 있는 API를 추가했습니다.

## 작업 배경

관련 이슈: S14P31A404-353\
참고 문서/기획: 모바일 비밀번호 변경 화면 API 요청

## 주요 변경 사항

- `PATCH /api/v1/users/me/password` API 추가
- 현재 비밀번호 검증 및 새 비밀번호 동일 여부 차단 로직 추가
- 새 비밀번호를 회원가입과 동일한 정책으로 검증하도록 요청 DTO 추가
- 비밀번호 변경 성공 시 `204 No Content` 반환
- 비밀번호 변경 서비스 테스트 추가

## 스크린샷/로그

- UI 변경 없음
- API 변경으로 Swagger 또는 API 응답 캡처 첨부 예정

## 테스트 결과

- `./gradlew.bat spotlessCheck --no-daemon` 통과
- `./gradlew.bat build -x test --no-daemon` 통과
- 참고: 전체 `test` 실행은 기존 `TranslationServiceImplTest`가 현재 develop의 번역 서비스 시그니처와 맞지 않아 `compileTestJava` 단계에서 실패 가능성이 있습니다.

- [x] 불필요한 로그/디버그 코드 제거
- [x] 로컬 빌드/런 정상

## 메모

- 비밀번호 변경 성공 후 기존 Access Token/Refresh Token은 유지합니다.
- 현재 비밀번호 불일치와 새 비밀번호 동일 케이스는 각각 전용 에러 코드로 구분했습니다.
  - `USER_CURRENT_PASSWORD_MISMATCH`
  - `USER_NEW_PASSWORD_SAME_AS_CURRENT`
- 추가로 비밀번호 허용 특수문자를 `!@#$%^&*`로 제한했으며, 모바일 회원가입 화면의 1차 검증 정책도 동일하게 맞춰야 합니다.

백엔드 비밀번호 정책이 `!@#$%^&*`만 허용하는 것으로 확정된 것으로 이해했습니다. :grin:

- `PWD_SPECIALS = "(?=.*[!@#\$%^&*])"`
- `PWD_ALLOWED_CHARS = "[A-Za-z\\d!@#\$%^&*]"`

이후 비밀번호 변경 화면 구현 시에도 같은 정책을 재사용하겠습니다.
