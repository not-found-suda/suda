## Description

## 요약
- 모바일 인증 API 최신 명세에 맞춰 회원가입 이름 입력과 Refresh Token Body 기반 세션 저장/갱신 흐름을 반영했습니다.

## 작업 배경
- 관련 이슈: #S14P31A404-294
- 참고 문서/기획: 백엔드 인증 API 최신 명세

## 주요 변경 사항
- 회원가입 요청/화면에 보호자 이름 `name` 필드를 추가하고 입력 검증을 적용했습니다.
- 로그인 응답의 `refreshToken`을 저장하도록 `saveTokens(accessToken, refreshToken)` 흐름으로 변경했습니다.
- Body 기반 Refresh API 요청/응답 DTO와 `RetrofitRefreshTokenClient`를 구현했습니다.
- Refresh 성공 시 access/refresh token을 함께 갱신하고, 실패/저장 오류 시 세션을 안전하게 만료하도록 보강했습니다.
- 스코프 아웃: 백엔드 Logout API 연동은 후속 티켓에서 진행합니다.

## 스크린샷/로그
- UI 변경: 회원가입 화면에 보호자 이름 입력 필드 추가
- 인증 로그: 토큰 값은 로깅하지 않으며, refresh 실패/저장 실패는 고정 메시지로만 기록

## 메모
- Refresh Token은 Cookie가 아닌 Body 기반 전달 방식으로 반영했습니다.
- Refresh API는 `NoAuth` Retrofit을 사용해 Authenticator 재귀 호출 위험을 피했습니다.
- Logout API 연동은 refresh token 저장 구조를 기반으로 후속 작업에서 진행하면 됩니다.
