## Description

## 요약

* Gemini 문맥 보정 실패 시에도 Clova STT 결과를 반환하도록 fallback 처리 추가

## 작업 배경

* 관련 이슈: S14P31A404-287
* 참고 문서/기획:

  * 모바일 연동 시 Gemini 보정 실패로 STT API가 502 반환되는 문제 발생
  * Clova STT는 정상 동작하나 Gemini 단계 실패가 전체 API 실패로 전파됨

## 주요 변경 사항

* Gemini 보정 실패 시 BusinessException throw 제거 → fallback 처리로 변경
* recognizedText 기반으로 correctedText 반환 (`corrected=false`)
* Gemini 호출 실패 시 로그 보강 (status / headers / response body 일부)
* Accept: application/json 헤더 추가로 응답 파싱 안정성 개선

## 스크린샷/로그

* Gemini 실패 시 (API Key 오타로 테스트)

```json
{
  "recognizedText": "엄마 배고파 뭐 줄까",
  "correctedText": "엄마 배고파 뭐 줄까",
  "corrected": false,
  "confidence": null,
  "locale": "ko-KR"
}
```

* Gemini 성공 시

```json
{
  "recognizedText": "엄마 배고파 뭐 줄까",
  "correctedText": "엄마 배고파. 뭐 줄까",
  "corrected": true,
  "confidence": null,
  "locale": "ko-KR"
}
```
