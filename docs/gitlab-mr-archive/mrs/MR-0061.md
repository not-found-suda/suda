```text
**P2: 서버 STT 요청 완료 후 상태 검증 및 파일 정리 필요**

Cloud STT 녹음 루프는 `cloudSttJob`으로 관리되고 있지만, 실제 서버 업로드는 `performCloudStt()` 내부의 별도 coroutine에서 실행되고 있어요. 이 구조에서는 세션 종료 이후 서버 응답이 도착했을 때, 이미 끝난 세션의 STT 결과가 다시 화면에 반영될 가능성이 있습니다.

또한 업로드에 사용한 임시 녹음 파일이 요청 완료 후에도 남아 있을 수 있는데, 사용자 음성이 포함된 파일인 만큼 성공/실패 여부와 관계없이 정리해두면 더 안전할 것 같아요.

아래 방향으로 개선하면 좋을 것 같습니다.

- 서버 STT 요청 Job을 별도로 관리해 `stopSession()`에서 함께 취소
- 응답 처리 전에 세션이 여전히 Active 상태인지 확인
- `finally` 블록에서 임시 파일을 삭제해 데이터가 남지 않도록 처리
```
