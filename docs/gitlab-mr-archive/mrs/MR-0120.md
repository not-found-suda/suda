## Description

## 요약
- 네이버 OAuth 로그인 방식을 PKCE/Custom Tabs에서 Android SDK 기반 provider access token 방식으로 전환했습니다.

## 작업 배경
- 관련 이슈: #S14P31A404-233
- 참고 문서/기획:
  - 백엔드 네이버 로그인 API가 providerAccessToken 전달 방식으로 변경됨
  - POST /api/v1/auth/oauth/naver
  - Request: { "providerAccessToken": "..." }
  - Response: 일반 로그인과 동일한 accessToken/refreshToken 응답

## 주요 변경 사항
- 네이버 Android SDK 의존성 및 BuildConfig 환경값을 추가했습니다.
- 기존 PKCE/Custom Tabs 딥링크 처리 코드를 제거했습니다.
- 네이버 SDK에서 받은 provider access token을 백엔드 로그인 API로 전달하도록 수정했습니다.
- 기존 이메일 로그인 UI/토큰 저장/아이 선택 라우팅 흐름을 유지한 채 네이버 로그인만 연결했습니다.
- 스코프 아웃: 네이버 개발자 콘솔 테스트 계정/패키지 등록 문제는 백엔드/콘솔 설정에서 확인 필요합니다.

## 스크린샷/로그
- UI는 기존 로그인 화면을 유지하며, 네이버 로그인 버튼이 활성화됩니다.
- 로컬 검증 로그:
  - ktlintCheck / detekt / lintDebug / testDebugUnitTest / assembleDebug 통과

## 테스트 결과 (선택사항)
- [x] 단위 테스트 작성 / 통과 여부
- [ ] API 응답/로그 캡처 (스크린샷 첨부)

## 작성자 체크리스트
- [x] Secret/Env/개인정보가 포함되지 않음
- [x] 불필요한 로그/디버그 코드 제거
- [x] 로컬 빌드/런 정상

## 메모
- `.env`에 NAVER_CLIENT_ID, NAVER_CLIENT_SECRET, NAVER_CLIENT_NAME 값이 필요합니다.
- 모바일에는 Client Secret을 코드에 하드코딩하지 않고 BuildConfig로 주입합니다.
- 현재 네이버 인증 후 백엔드 요청까지는 진행되며, 네이버 개발자 콘솔의 Android 패키지/테스트 계정 등록 상태에 따라 provider token 검증이 실패할 수 있습니다.
- 기존 PKCE용 redirect intent-filter와 MainActivity redirect 처리 로직은 SDK 방식 전환으로 제거했습니다.

**\[P2\] 네이버 로그인 실패 응답의 code/traceId를 버리고 있습니다.**
