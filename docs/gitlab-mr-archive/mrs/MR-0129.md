## 요약
- 학습 단어장 이미지/음성 에셋을 S3에 업로드하고, DB에는 S3 object key 기반으로 저장되도록 마이그레이션을 추가했습니다.


## 작업 배경
- 관련 이슈: S14P31A404-301
- 참고 문서/기획:
  - 카테고리/난이도별 단어장 학습 기능
  - 모바일 단어장 화면에서 이미지 표시 및 음성 재생을 위한 asset key 데이터 필요


## 주요 변경 사항
- 동물/음식/가족/색깔/탈것 카테고리의 단어별 이미지 및 음성 파일을 S3 경로 기준으로 정리
- learn 테이블에 image_url, audio_url 값을 S3 URL이 아닌 object key 형태로 저장하는 마이그레이션 추가
- category_id, difficulty, sort_order를 고려하여 카테고리/난이도별 단어 데이터 추가 및 갱신
- 기존 잘못 저장된 일부 동물 단어 asset key 경로 수정
- 이미 적용된 Flyway migration 파일은 수정하지 않고 신규 migration 파일로 데이터 반영


## 스크린샷/로그
- API/DB 데이터 변경 작업으로 별도 UI 스크린샷 없음
- S3 업로드 경로 예시:
  - `learn/words/animals/normal/...`
  - `learn/words/food/easy/...`
  - `learn/words/family/easy/...`
  - `learn/words/color/hard/...`
  - `learn/words/vehicle/normal/...`

## 메모
- DB에는 S3 전체 URL이 아닌 object key만 저장합니다.
  - 예: `learn/words/food/easy/apple.png`
  - 예: `learn/words/food/easy/apple.mp3`
- 실제 URL 조합은 백엔드의 AssetUrlResolver/CloudFront 변환 로직을 통해 처리됩니다.
- Flyway checksum mismatch 방지를 위해 이미 실행된 migration 파일은 수정하지 않고, 신규 migration 파일을 추가하는 방식으로 반영했습니다.
- 업로드된 S3 파일명과 DB asset key가 일치해야 모바일에서 이미지 로딩 및 음성 재생이 정상 동작합니다.
