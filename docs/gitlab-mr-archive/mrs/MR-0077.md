## Description

## 요약
- 로그인/회원가입 로컬 API 연동을 위해 인증 API 경로와 debug HTTP 통신 설정을 수정

## 작업 배경
- 관련 이슈: #S14P31A404-288
- 참고 문서/기획: 로컬 백엔드 연동 테스트, 인증 API baseUrl 설정

## 주요 변경 사항
- debug 빌드에서만 HTTP cleartext 통신을 허용하도록 debug Manifest 설정 추가
- Login API endpoint의 중복 `api/` 경로 제거
- Signup API endpoint의 중복 `api/` 경로 제거
- `BASE_URL=http://.../api/` 기준으로 `/api/v1/auth/login`, `/api/v1/auth/signup` 요청이 나가도록 수정
- 변경하지 않은 것 / 스코프 아웃: release 빌드 cleartext 허용, 인증 응답/토큰 정책 변경

## 메모
- `android:usesCleartextTraffic="true"`는 `app/src/debug/AndroidManifest.xml`에만 적용되어 debug 빌드에서만 로컬 HTTP 통신을 허용합니다.
- `BASE_URL`이 `/api/`로 끝나는 현재 구조를 유지하고, Retrofit service endpoint에서 `api/` prefix만 제거했습니다.
- 실제 기기 테스트 시 PC IP 기반 baseUrl은 같은 네트워크 접근 가능 여부와 백엔드 bind address 확인이 필요합니다.
