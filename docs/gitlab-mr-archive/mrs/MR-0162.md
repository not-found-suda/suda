## 요약

* 부모 음성 기반 TTS 선택 기능 추가 및 비로그인/로그인 사용자별 음성 적용 로직 구현

---

## 작업 배경

* 관련 이슈: S14P31A404-365
* 참고 문서/기획: CODA 양방향 소통 기능 개선 기획

부모가 수어를 통해 전달하는 내용을 아이가 음성으로 들을 때, 기본 TTS 음성이 아닌 **부모 느낌의 목소리로 전달되도록 개선**하기 위해 기능을 추가했습니다. 또한 로그인 여부에 따라 기본 음성과 개인화된 음성을 구분하여 적용하도록 설계했습니다.

---

## 주요 변경 사항

* 사용자 TTS speaker 선택 기능 추가 (DB 컬럼 및 enum 도입)
* sign-to-speech API에서 사용자별 speaker 적용 로직 구현
* 비로그인 사용자 → 기본 음성(vara), 로그인 사용자 → 선택 음성 적용(엄마 목소리 2, 아빠 목소리 2)
* Clova TTS 연동 로직 개선 (emotion 제거 및 안정화)
* Swagger에서 인증 여부와 관계없이 테스트 가능하도록 구조 개선

---

* TTS 변환 성공 로그

* 최종 적용 speaker:

  * vara (기본-엄마)
  * vgoeun (엄마)
  * vdonghyun (아빠)
  * neunwoo (아빠)
* 비로그인 사용자도 TTS 기능은 사용 가능하되, 기본 음성만 제공하도록 설계
* 이후 확장 시 speaker 목록 관리 API 또는 외부 설정화 고려 가능
