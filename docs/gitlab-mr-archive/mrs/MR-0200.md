```text
\[P2\] FastAPI 추론 info 로그가 실제 컨테이너 로그에 남지 않을 수 있습니다

이번 변경에서 logger = logging.getLogger("sign_ai")로 추론 시작/성공 로그를 info 레벨로 남기고 있습니다. 그런데 Dockerfile은 기본 uvicorn app.main:app ... 실행만 하고 있고, 이 브랜치에는 sign_ai logger의 level/handler 설정이 없습니다. Python 기본 root logger는 WARNING 기준이라, 별도 설정이 없으면 관측성 목적으로 추가한 request start/success info 로그가 stdout 컨테이너 로그에 찍히지 않을 수 있습니다. uvicorn.error logger를 사용하거나, 앱 시작 시 sign_ai logger가 INFO 이상을 stdout으로 내보내도록 logging 설정을 추가하는 게 좋겠습니다.
```

### nej0720 at 2026-05-18T21:37:45.992+09:00 (comment)

```text
수정 내용:

- FastAPI 앱 시작 시 sign_ai logger를 INFO 레벨로 설정하도록 추가했습니다.
- sign_ai logger에 StreamHandler(sys.stdout)를 명시적으로 붙여 request start/success 로그가 Docker 컨테이너 로그에 남도록 보완했습니다.
- logger.propagate=false로 설정해 다른 logger로 인한 중복 출력 가능성을 줄였습니다.

이제 uvicorn 기본 실행 환경에서도 수어 추론 start/success info 로그가 누락되지 않고 컨테이너 로그에서 확인될 수 있습니다.
```
