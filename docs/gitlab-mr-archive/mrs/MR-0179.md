* LLM JSON 응답 파싱 및 단어 빈도/표현 유형/요약 데이터 집계 로직 추가
* 조회 쿼리 null 파라미터 문제 해결 및 전체 기간 조회 기본값 처리

```text
소통 발화 분석 리포트 API 구조는 모바일 연동에 충분해 보입니다. :grin: :grin: 

한 가지 확인만 부탁드립니다.
현재 `CommunicationSessionAnalysisService.analyzeSession()`이 하나의 트랜잭션 안에서 `markProcessing()`과 LLM 호출/완료 처리를 함께 수행하는 구조라, 모바일에서 `PROCESSING` 상태를 실제로 관측하기 어려울 수 있어 보입니다.

`PENDING -> COMPLETED/FAILED/EMPTY`로만 보이는 구조여도 모바일은 `PENDING`을 분석 중으로 처리하면 되므로 치명적 문제는 아니지만, 명세상 `PROCESSING`을 내려줄 의도라면 별도 트랜잭션 커밋 여부를 한 번 확인 부탁드립니다.
```
