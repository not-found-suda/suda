```text
**P4: 목록 재시도 요청이 겹치지 않도록 관리하면 더 좋겠습니다**

`ChildProfileSelectViewModel`에서 `retry()`를 누를 때마다 `loadProfiles()`가 새 `coroutine`을 실행합니다. 현재 `FakeRepository`에서는 큰 문제가 없지만, 실제 API로 바뀐 뒤 사용자가 재시도를 여러 번 누르면 이전 요청과 새 요청이 겹칠 수 있습니다.

크게 위험한 문제는 아니지만, loadJob을 따로 관리해서 기존 요청을 취소하거나 Loading 중에는 재시도 버튼을 막아두면 더 안정적일 것 같습니다.
```
