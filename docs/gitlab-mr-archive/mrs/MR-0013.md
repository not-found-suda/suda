## 요약
- 온디바이스 및 무상태(Stateless) 아키텍처에 맞춘 API 명세서(`api-spec.md`)를 작성했습니다.

## 작업 배경
- 관련 이슈: #S14P31A404-130
- 참고 문서/기획: 시스템 데이터 흐름도(`system-flow.md`)
- 기존 서버 중심의 무거운 세션 및 웹소켓 API를 덜어내고, MVP 단계에 맞춘 가벼운 RESTful API 규격을 정의하여 프론트/백엔드 개발의 기준점을 세우고자 합니다.

## 주요 변경 사항
- `docs/api/api-spec.md` 파일 신규 생성 및 Base URL(`https://api.suda.com/api/v1`) 정의
- 음성->텍스트 API 응답에 UI 토글 분기를 위한 **듀얼 응답(원본 `raw_text` + 보정본 `corrected_text`) JSON 구조** 적용
- 에러 응답 규격으로 글로벌 표준인 **RFC 9457 (Problem Details for HTTP APIs)** 포맷 도입
- 변경하지 않은 것: 오번역 신고 및 세션 DB 저장 등은 MVP 스코프 아웃으로 판단하여 향후 확장 계획으로 분리했습니다.
