# 2026-05-18 V7 시연 문장 연속 인식 성능 점검

## 목적

- V7 온디바이스 수어 인식 모델 기준으로 발표 시연에 사용할 수 있는 안정적인 단어와 gloss 문장을 선별한다.
- 단독 단어 인식뿐 아니라 실제 연속 수어 입력에서 단어가 순서대로 인식되고 최종 문장까지 안정적으로 확정되는지 확인한다.
- 최종 gloss 문장이 한국어 문법 체계로 변환되고, 채팅/TTS 출력 흐름까지 자연스럽게 이어질 수 있는 시연 후보를 정리한다.
- 수어-한국어 변환 결과는 농인 부모 또는 보호자가 3~5세 아이에게 직접 말하는 문장으로 정리한다.
- 오늘 이후 진행하는 테스트 결과와 판단 근거는 이 문서에 계속 누적한다.

## 현재 테스트 기준

- 대상 모델: V7 TCN TFLite 온디바이스 모델
- 테스트 대상: 시연 대상 단어 116개
- 앱 모드: 자동 모드 기준으로 테스트
- 추론 어댑터: `TfliteSignInferenceAdapter`
- full segment inference 지원 여부: `false`
- hand forward-fill: 비활성화
- 모델 입력: RAW landmark sequence, `[1, 30, 332]`
- 연속 문장 기준: 3개 이상 gloss 단어 조합
- 주요 확인 항목: 단어별 인식 안정성, 연속 입력 시 단어 전환 안정성, 최종 문장 확정 결과, 평서문/의문문 오판정 여부

## 운영 원칙

- 단어 단독 테스트를 먼저 수행한 뒤, 안정 단어만 조합해 3단어 이상 문장 테스트를 진행한다.
- 단어별 기본 테스트 횟수는 2회로 한다.
- 1회 정답, 1회 오답인 단어는 재검증 대상으로 분류한다.
- 2회 모두 오답이거나 동일 단어로 고정 오인식되는 경우 시연 후보에서 제외하고 known issue로 기록한다.
- 최종 gloss가 맞더라도 문장 타입이 평서문/의문문으로 흔들리는 경우 별도 이슈로 기록한다.

## 오후 테스트 관찰

- 연속 인식 자체는 정상 동작한다.
- `엄마 아프다 치료`, `엄마 서점 서다`는 2회 모두 목표 gloss 순서대로 인식되었다.
- `시험 모르다 위로`는 `시험`이 2회 모두 `손자`로 인식되어 시연 후보에서 제외하는 것이 안전하다.
- 평서문으로 의도한 문장이 2회차에서 의문문으로 확정된 사례가 있었다.
- gloss 인식 안정성과 문장 타입 판정 안정성은 분리해서 관리해야 한다.

## 2026-05-19 번역 전략 및 시연 rule 정리

### 현재 구조 확인

- 온디바이스 번역 모델은 `Qwen2.5 1.5B Instruct` LiteRT-LM 모델이다.
- 앱의 온디바이스 번역 흐름은 단순히 Qwen만 호출하는 구조가 아니다.
- 현재 흐름은 대략 다음 순서로 동작한다.

```text
인식된 gloss
-> ConversationViewModel 데모용 rule 확인
-> LocalSignSentenceGenerator known pattern rule 확인
-> Qwen SLLM 번역
-> Qwen 실패 또는 빈 응답 시 local fallback
```

- 따라서 앞단 rule에 걸리는 문장은 Qwen 프롬프트를 수정해도 결과가 바뀌지 않는다.
- 시연 핵심 문장은 `ConversationViewModel`에서 평서문/의문문 문장 타입을 함께 반영해 고정한다.
- Qwen 쪽에도 같은 방향의 override와 few-shot을 둔다. 다만 실제 대화 화면에서는 `ConversationViewModel` rule이 먼저 적용된다.

### 서비스 목적 기준

- 수어를 입력하는 사람은 농인 부모 또는 보호자다.
- 출력 문장을 듣는 사람은 3~5세 아이다.
- 따라서 번역 결과는 부모가 아이에게 직접 말하는 짧고 따뜻한 문장이어야 한다.
- 현재 모델에는 `아빠` 라벨이 없으므로, 시연에서는 `엄마` gloss를 보호자/아빠 발화로 해석한다.
- 출력에서 보호자 호칭이 필요하면 `엄마` 대신 `아빠`를 사용한다.
- 수어 동작 방향성을 앱에서 안정적으로 알 수 없으므로, 방향성은 기본적으로 `부모/보호자 -> 아이`로 둔다.
- `주다`, `받다`, `돕다`, `위로`, `치료`처럼 방향성이 중요한 단어는 보호자가 아이에게 해 주는 말이나 행동으로 보정한다.

### rule 기반으로 고정할 문장

- 데모에서 반드시 성공해야 하는 핵심 문장이다.
- Qwen이 방향을 틀리거나 제3자 설명으로 만들 가능성이 있는 문장은 rule로 고정하는 것이 안전하다.
- 이는 모든 문장을 하드코딩하는 것이 아니라, 어린 아이에게 전달되는 문장의 방향성과 안전성을 보장하는 보정 레이어로 설명한다.

### SLLM에 맡길 문장

- rule에 없는 조합은 Qwen SLLM에 맡긴다.
- 단, Qwen 결과가 아이 대상 발화가 아니라 제3자 설명으로 나오거나, 보호자->아이 방향이 뒤집히면 rule 후보로 승격한다.
- 입력 gloss에 없는 단어를 과하게 추가하거나, 의학적 진단/원인/처방을 추측하는 출력은 fallback 또는 rule 보정 대상으로 본다.

예시 후보:

- `몸 아프다`
- `바람 조심`
- `손 조심`
- `이불 자다`
- `아프다 치료`

### 발표/시연 설명 관점

- 일부 rule 기반 처리는 하드코딩처럼 보일 수 있으므로, "정답 문장 암기"가 아니라 "안전/방향성 보정 레이어"로 설명한다.
- 대부분의 조합은 Qwen 온디바이스 SLLM이 번역하고, 방향성이 중요한 일부 표현만 규칙 기반으로 보정한다고 설명한다.
- 실제 서비스에서도 LLM 앞뒤에 입력 정규화, guardrail, fallback, safety rule을 두는 것은 자연스러운 설계다.
- 전부 rule 기반으로 만드는 것은 지양하고, 데모 핵심 문장과 안전 관련 문장만 고정한다.
