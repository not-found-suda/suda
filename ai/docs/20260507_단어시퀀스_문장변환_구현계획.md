# 단어 시퀀스 생성 및 한국어 문장 변환 구현 계획

## 1. 목표

수어 단어 인식 모델이 실시간으로 출력하는 단어 후보를 바로 문장 변환에 사용하지 않고, 안정적으로 확정된 단어만 모아 gloss 형태의 단어 시퀀스를 만든다. 이후 이 gloss 시퀀스를 GPT 계열 모델에 전달해, 농인 부모가 3~5세 CODA 아이에게 말하는 듯한 자연스러운 한국어 문장으로 변환한다.

현재 최종 모델 인식 단어:

```text
기차, 장난감, 놀다, 가다, 일어나다, 병원, 조심
```

전체 흐름:

```text
실시간 TFLite 추론
→ 단어 후보 안정화
→ 확정 단어 이벤트 생성
→ 문장 버퍼에 단어 누적
→ 중복/노이즈 단어 제거
→ 양손 모두 사라짐으로 문장 끝 판단
→ gloss 시퀀스를 GPT에 전달
→ 아이에게 말하는 자연스러운 한국어 문장 생성
```

## 2. 문장 시작 기준

문장 시작은 **첫 번째 단어가 확정되는 순간**으로 본다.

손이 화면에 보이기 시작한 시점만으로는 문장 시작으로 보지 않는다. 손을 준비 자세로 올리거나 위치를 잡는 동안 잘못된 후보가 나올 수 있기 때문이다.

상태 전이:

```text
IDLE
  첫 번째 확정 단어 발생
  → COLLECTING
  → sentenceBuffer 초기화 후 첫 단어 추가
```

예:

```text
확정 단어: 병원
sentenceBuffer = ["병원"]
문장 상태 = COLLECTING
```

## 3. 단어 확정 기준

초기 기준은 현재 inference 코드와 동일하게 사용한다.

```text
confidence threshold = 0.75
margin threshold = 0.08
prediction smoothing window = 6
stable minimum count = 4
minimum frames for prediction = 15
```

확정 조건:

```text
top1_confidence >= 0.75
top1_confidence - top2_confidence >= 0.08
최근 6회 예측 중 같은 단어가 최소 4회 이상 등장
```

이 조건을 통과한 단어만 `confirmedWord`로 취급하고 문장 버퍼에 넣는다.

이 값들은 실제 모바일 테스트 후 조정 가능하다.

```text
오인식이 많음 → confidence / margin / stable minimum count 상향
반응이 너무 느림 → stable minimum count 하향 또는 minimum frames 하향
```

## 4. 중복 단어 제거 전략

중복 제거는 단순히 인접한 같은 단어만 제거하는 것으로는 부족하다. 실시간 추론에서는 중간에 짧은 노이즈 단어가 끼어들 수 있기 때문이다.

예:

```text
raw confirmed words:
["가다", "가다", "기차", "가다", "가다"]
```

이 경우 `기차`는 짧게 튄 노이즈로 보고 최종 시퀀스는 다음처럼 처리한다.

```text
["가다"]
```

따라서 후처리는 두 단계로 한다.

### 4.1 1단계: 짧은 노이즈 제거

패턴이 `A, B, A`이고, `B`가 짧은 시간 안에 1회만 등장한 경우 `B`를 제거한다.

```text
["가다", "기차", "가다"] → ["가다", "가다"]
["병원", "장난감", "병원"] → ["병원", "병원"]
```

권장 조건:

```text
B가 1회만 등장
A와 B, B와 A 사이 시간 간격이 각각 1.0초 이하
B의 confidence가 A보다 낮거나, B의 지속 시간이 짧음
```

단순 데모 구현에서는 우선 아래 조건만 적용해도 된다.

```text
if previousWord == nextWord and middleWord != previousWord:
    remove middleWord
```

### 4.2 2단계: 연속 중복 제거

노이즈 제거 후 연속으로 같은 단어가 이어지면 하나로 합친다.

```text
["가다", "가다"] → ["가다"]
["병원", "병원", "가다", "가다"] → ["병원", "가다"]
```

최종 예:

```text
입력: ["가다", "가다", "기차", "가다", "가다"]

1단계 노이즈 제거:
["가다", "가다", "가다", "가다"]

2단계 연속 중복 제거:
["가다"]
```

## 5. 문장 끝 기준

문장 끝은 **양손이 모두 화면에서 사라진 상태가 1.2~1.5초 이상 지속될 때**로 판단한다.

중요: 한 손만 사라진 상태는 문장 끝으로 보지 않는다. 수어 동작 중 한 손만 잠시 검출되지 않거나 화면 밖으로 나갈 수 있기 때문이다.

손 감지 기준:

```text
leftHandVisible = left hand landmarks 존재
rightHandVisible = right hand landmarks 존재
bothHandsMissing = !leftHandVisible && !rightHandVisible
```

문장 끝 후보:

```text
bothHandsMissing 상태가 1.2초 이상 지속
```

권장값:

```text
초기값: 1.3초
조정 범위: 1.2초 ~ 1.5초
```

상태 전이:

```text
COLLECTING
  bothHandsMissingDuration >= 1.3초
  → READY_TO_CONVERT
  → sentenceBuffer 후처리
  → GPT 변환 요청
```

## 6. GPT에 넘길 데이터 형태

GPT에는 영상이나 landmark 좌표를 넘기지 않는다. 모델에서 확정된 단어 시퀀스만 넘긴다.

권장 JSON 형식:

```json
{
  "gloss": ["병원", "가다", "조심"],
  "sentenceType": "평서문",
  "availableWords": ["기차", "장난감", "놀다", "가다", "일어나다", "병원", "조심"],
  "tone": "농인 부모가 3~5세 CODA 아이에게 말하는 따뜻하고 쉬운 말투"
}
```

`availableWords`는 GPT가 입력 gloss에 없는 의미를 만들어내지 않도록 제한하기 위한 단어 목록이다. 즉, 모델이 실제로 인식할 수 있는 단어 범위를 알려주는 guardrail 역할을 한다.

좋은 변환:

```text
["병원", "가다"] → "병원에 가자."
```

나쁜 변환:

```text
["병원", "가다"] → "엄마랑 병원에 가서 주사를 맞자."
```

위 나쁜 예시는 `엄마`, `주사`라는 입력에 없는 의미를 추가했기 때문에 피해야 한다.

평서문과 의문문도 **같은 JSON 형태**로 보낸다. 달라지는 것은 `sentenceType` 값뿐이다.

평서문:

```json
{
  "gloss": ["기차", "가다"],
  "sentenceType": "평서문",
  "availableWords": ["기차", "장난감", "놀다", "가다", "일어나다", "병원", "조심"],
  "tone": "농인 부모가 3~5세 CODA 아이에게 말하는 따뜻하고 쉬운 말투"
}
```

의문문:

```json
{
  "gloss": ["기차", "가다"],
  "sentenceType": "의문문",
  "availableWords": ["기차", "장난감", "놀다", "가다", "일어나다", "병원", "조심"],
  "tone": "농인 부모가 3~5세 CODA 아이에게 말하는 따뜻하고 쉬운 말투"
}
```

## 7. GPT 프롬프트 초안

목표 말투는 **농인 부모가 3~5세 CODA 아이에게 말하는 자연스럽고 쉬운 한국어**다. 너무 딱딱한 번역문이 아니라, 짧고 따뜻하며 아이가 이해하기 쉬운 문장으로 만든다.

프롬프트 초안:

```text
너는 한국 수어 gloss 단어 시퀀스를 자연스러운 한국어 문장으로 바꾸는 변환기다.

상황:
- 말하는 사람은 농인 부모다.
- 듣는 사람은 3~5세 CODA 아이다.
- 따라서 문장은 짧고, 쉽고, 따뜻한 말투여야 한다.

규칙:
1. 입력 gloss에 있는 단어의 의미만 사용한다.
2. 입력에 없는 새로운 사람, 장소, 사물, 사건을 추가하지 않는다.
3. 사용 가능한 단어 목록 밖의 핵심 의미를 새로 만들지 않는다.
4. 단어 순서는 가능한 유지한다.
5. 한국어 조사와 어미는 자연스럽게 보완한다.
6. sentenceType이 "의문문"이면 아이에게 묻는 말투로 만들고 물음표로 끝낸다.
7. sentenceType이 "평서문"이면 부드러운 평서문 또는 권유문으로 만든다.
8. 출력은 한국어 문장 하나만 반환한다.
9. 설명, 후보, 따옴표, JSON 없이 문장만 출력한다.

사용 가능한 단어:
{availableWords}

입력 gloss:
{gloss}

문장 유형:
{sentenceType}

출력:
```

예시:

```text
gloss = ["병원", "가다"]
sentenceType = "평서문"
출력 = 병원에 가자.
```

```text
gloss = ["기차", "가다"]
sentenceType = "의문문"
출력 = 기차 타러 갈까?
```

```text
gloss = ["장난감", "조심"]
sentenceType = "평서문"
출력 = 장난감 조심하자.
```

## 8. 규칙 기반 변환과 GPT 사용 전략

당장은 GPT를 테스트해보되, 데모 안정성을 위해 자주 쓰는 조합은 규칙 기반 결과를 먼저 준비하는 것이 좋다.

규칙 기반 예시:

```json
{
  "기차,가다": "기차 타러 가자.",
  "병원,가다": "병원에 가자.",
  "장난감,놀다": "장난감으로 놀자.",
  "장난감,조심": "장난감 조심하자.",
  "기차,조심": "기차 조심하자.",
  "병원,가다,조심": "병원에 가서 조심하자.",
  "놀다,일어나다": "놀다가 일어나자."
}
```

의문문이면 규칙 결과를 의문문 말투로 바꾼다.

```json
{
  "기차,가다": "기차 타러 갈까?",
  "병원,가다": "병원에 갈까?",
  "장난감,놀다": "장난감으로 놀까?"
}
```

권장 처리:

```text
if ruleBasedMapping exists:
    use ruleBasedSentence
else:
    call GPT
```

나중에 단어 수가 늘어나면 GPT 사용 비중을 늘릴 수 있다.

## 9. 상태 머신 설계

```text
IDLE
  - 문장 대기 상태
  - 확정 단어가 발생하면 COLLECTING으로 이동

COLLECTING
  - 확정 단어를 sentenceBuffer에 누적
  - 연속 중복 단어는 무시
  - 짧게 튄 노이즈 단어는 후보 상태로 보류
  - 양손 모두 사라진 시간을 측정

READY_TO_CONVERT
  - 양손 모두 사라짐이 1.3초 이상 지속된 상태
  - sentenceBuffer 후처리 수행
  - 빈 버퍼면 IDLE로 복귀
  - 단어가 있으면 CONVERTING으로 이동

CONVERTING
  - 규칙 기반 변환 또는 GPT API 호출
  - 변환 결과를 화면에 표시

DONE
  - 최종 문장 표시
  - 다음 손 동작이 시작되면 새 문장으로 IDLE 또는 COLLECTING 복귀
```

## 10. 구현 의사코드

```kotlin
enum class SentenceState {
    IDLE,
    COLLECTING,
    READY_TO_CONVERT,
    CONVERTING,
    DONE
}

data class ConfirmedWordEvent(
    val label: String,
    val confidence: Float,
    val margin: Float,
    val timestamp: Long
)

val sentenceBuffer = mutableListOf<ConfirmedWordEvent>()
var state = SentenceState.IDLE
var bothHandsMissingStartTime: Long? = null
val sentenceEndMs = 1300L

fun onFrameResult(
    leftHandVisible: Boolean,
    rightHandVisible: Boolean,
    confirmedWord: ConfirmedWordEvent?
) {
    val now = System.currentTimeMillis()
    val bothHandsMissing = !leftHandVisible && !rightHandVisible

    if (confirmedWord != null) {
        bothHandsMissingStartTime = null

        if (state == SentenceState.IDLE || state == SentenceState.DONE) {
            sentenceBuffer.clear()
            state = SentenceState.COLLECTING
        }

        appendWordWithDedup(confirmedWord)
    }

    if (state == SentenceState.COLLECTING) {
        if (bothHandsMissing) {
            if (bothHandsMissingStartTime == null) {
                bothHandsMissingStartTime = now
            }

            val missingDuration = now - bothHandsMissingStartTime!!
            if (missingDuration >= sentenceEndMs) {
                state = SentenceState.READY_TO_CONVERT
                finalizeSentence()
            }
        } else {
            bothHandsMissingStartTime = null
        }
    }
}
```

중복 제거:

```kotlin
fun appendWordWithDedup(event: ConfirmedWordEvent) {
    val last = sentenceBuffer.lastOrNull()

    // 1. 바로 직전 단어와 같으면 중복으로 무시
    if (last != null && last.label == event.label) {
        return
    }

    sentenceBuffer.add(event)

    // 2. A-B-A 형태의 짧은 노이즈 제거
    removeShortBridgeNoise()
}

fun removeShortBridgeNoise() {
    if (sentenceBuffer.size < 3) return

    val n = sentenceBuffer.size
    val a = sentenceBuffer[n - 3]
    val b = sentenceBuffer[n - 2]
    val c = sentenceBuffer[n - 1]

    val isBridgeNoise = a.label == c.label && b.label != a.label
    val shortGap = (b.timestamp - a.timestamp <= 1000L) &&
                   (c.timestamp - b.timestamp <= 1000L)

    if (isBridgeNoise && shortGap) {
        sentenceBuffer.removeAt(n - 2)
    }
}
```

문장 확정:

```kotlin
fun finalizeSentence() {
    val gloss = sentenceBuffer
        .map { it.label }
        .dedupeAdjacent()

    if (gloss.isEmpty()) {
        state = SentenceState.IDLE
        return
    }

    state = SentenceState.CONVERTING
    val sentence = convertGlossToKorean(
        gloss = gloss,
        sentenceType = currentSentenceType
    )

    showSentence(sentence)
    state = SentenceState.DONE
}
```

## 12. 정리

현재 합의한 핵심 기준:

```text
1. 문장 시작 = 첫 번째 단어 확정 순간
2. 단어 확정 = confidence/margin/smoothing 기준 통과
3. 중복 제거 = 연속 중복 제거 + A-B-A 짧은 노이즈 제거
4. 문장 끝 = 양손 모두 사라진 상태가 1.2~1.5초 지속
5. GPT 입력 = gloss, sentenceType, availableWords를 같은 형식으로 전달
6. 문장 변환 말투 = 농인 부모가 3~5세 CODA 아이에게 말하는 쉬운 말투
7. 규칙 기반 변환을 우선 적용하고, 매핑이 없으면 GPT 호출
```
