# 2026-05-08 최신 모델, 영상 추론, 모바일 디버깅

## 작업 배경

7개 핵심 단어 모델을 모바일에서 안정적으로 사용하기 위해, 최종 후보 모델을 다시 학습하고 TFLite 변환 및 추론 검증 흐름을 점검했다. 전날 기준의 모바일 전달 자료를 만들었지만, 실제 테스트와 디버깅 과정에서 더 안정적인 최종 모델을 모바일 적용 후보로 갱신했다.

이번 작업의 중심은 PyTorch 모델 자체의 validation 수치보다, TFLite 변환 이후 웹캠과 휴대폰 촬영 영상에서 동일한 라벨과 확률 흐름으로 추론되는지 확인하는 것이었다.

## 최신 7단어 모델 정리

최종 후보 단어는 기존 시연 범위에 맞춰 아래 7개로 유지했다.

```text
기차, 장난감, 놀다, 가다, 일어나다, 병원, 조심
```

원본 학습 산출물 중 Git 저장소에 반영할 모바일 적용 파일은 아래처럼 정리했다.

```text
label_map.json
train_config.json
best_sign_model_float16.tflite
best_sign_model_float32.tflite
```

`label_map.json`은 모바일과 웹 추론에서 동일한 클래스 순서를 사용하기 위한 기준 파일로 두었다. `train_config.json`에는 학습 시 사용한 입력 차원, 프레임 길이, 클래스 구성, 샘플링 조건을 남겨 모델 교체 시 추적 가능하도록 했다.

Git 저장소에는 모바일 적용 경로에 맞춰 파일명을 단순화해 반영한다.

```text
ai/models/transformer_mobile/label_map.json
ai/models/transformer_mobile/train_config.json
ai/models/transformer_mobile/best_sign_model_float16.tflite
ai/models/transformer_mobile/best_sign_model_float32.tflite
```

루트의 Transformer 코드들은 5월 8일까지 수정된 최신본으로 정리했다. 특히 데이터 추출, 다운로드, 학습, 데이터셋 구성, TFLite 변환, 실시간 추론, holdout 평가, 영상 파일 추론 쪽을 현재 모바일 검증 흐름에 맞게 갱신했다.

Git 저장소에 반영한 코드 파일은 다음과 같다.


Git 저장소 파일명에 맞춰 내부 import도 함께 정리했다.

```text

```

## TFLite 변환 및 모바일 적용

TFLite 변환 시 모바일 추론에서 바로 확률값을 사용할 수 있도록 softmax 출력이 유지되는지 확인했다. 변환 대상 모델 디렉터리는 기본적으로 `7words_test_great`를 우선 사용하도록 정리했다.

모바일 적용용 산출물은 용량과 사용 목적을 고려해 아래 파일만 Git 저장소 관리 대상으로 삼았다.

```text
ai/models/transformer_mobile/label_map.json
ai/models/transformer_mobile/train_config.json
ai/models/transformer_mobile/best_sign_model_float16.tflite
ai/models/transformer_mobile/best_sign_model_float32.tflite
```

원본 PyTorch 모델인 `.pt` 파일은 저장소에 직접 올리지 않는 방향으로 정했다. ONNX 파일도 이번 최종 모델 폴더에는 포함하지 않고, 이후에도 저장소에는 올리지 않는 기준으로 둔다.

## 실시간 TFLite 추론 디버깅

`ai/src/sign2text_transformer/inference_realtime_tflite.py`는 모바일 적용 전 TFLite 모델의 실제 추론 결과를 웹캠 환경에서 확인하기 위한 스크립트로 정리했다. confidence threshold, top-2 margin, smoothing, 최소 프레임 수, 손이 사라질 때까지 예측을 잠그는 옵션 등을 환경변수로 조정할 수 있게 했다.

주요 디버깅 기준은 다음과 같다.

```text
CONFIDENCE_THRESHOLD
MARGIN_THRESHOLD
PREDICTION_SMOOTHING
STABLE_MIN_COUNT
MIN_FRAMES_FOR_PREDICTION
LOCK_ACTION_UNTIL_NO_HANDS
```

이를 통해 단일 프레임의 순간적인 오인식보다, 일정 구간 동안 안정적으로 반복되는 예측을 최종 결과로 사용하는 방향을 유지했다.

## 영상 파일 추론 추가

모바일에서 촬영한 영상 파일을 직접 검증하기 위해 `ai/src/sign2text_transformer/inference_video_tflite.py`를 추가했다. 이 스크립트는 휴대폰 또는 카메라로 녹화한 영상 파일을 입력받아 MediaPipe feature를 추출하고, TFLite 모델로 구간별 단어를 예측한다.

문장 영상처럼 여러 단어가 이어지는 경우를 고려해, 손이 사라지는 구간 또는 손 움직임이 멈추는 구간을 단어 경계 후보로 사용할 수 있게 했다.

주요 옵션은 다음과 같다.

```text
--segment-end-seconds
--stride
--min-word-windows
--ignore-edge-windows
--split-on-pause / --no-split-on-pause
--pause-seconds
--pause-motion-threshold
--debug-timestamps-ms
```

이 스크립트는 단어 단위 모델을 문장 영상에 적용할 때, 어떤 방식으로 단어 시퀀스를 만들 수 있는지 확인하기 위한 디버깅 도구로 사용한다.
