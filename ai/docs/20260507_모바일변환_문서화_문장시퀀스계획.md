# 2026-05-07 모바일 변환, 문서화, 문장 시퀀스 계획

## 작업 배경

7개 핵심 단어 모델이 실제 촬영 데이터 기준으로 어느 정도 안정화된 뒤, 최종 목표인 모바일 온디바이스 추론을 위해 TFLite 변환과 팀원 전달 문서를 정리했다. 또한 단어 단위 인식 이후, 여러 단어가 이어진 문장 영상에서 단어 시퀀스를 어떻게 구성할지 논의했다.

## 핵심 단어 모델

한 시점의 핵심 단어 모델은 다음 7개 단어를 대상으로 했다.

```text
기차, 장난감, 놀다, 가다, 일어나다, 병원, 조심
```

모델 폴더에는 PyTorch 모델, label map, train config, ONNX, TFLite 모델, 문서가 함께 들어갔다.

```text
best_sign_model_1.pt
best_sign_model_1.onnx
best_sign_model_1_float32.tflite
best_sign_model_1_float16.tflite
label_map_1.json
train_config_1.json
```

## TFLite 변환

변환 스크립트는 PyTorch 모델을 ONNX로 export한 뒤, TensorFlow SavedModel을 거쳐 TFLite로 변환하는 흐름을 사용했다.

중요한 수정은 softmax 누락 방지였다. 모바일에서는 output을 확률값으로 바로 사용하기 때문에, 변환 시 `SoftmaxWrapper`를 통해 `return_probs=True`로 모델을 감싸도록 했다.

결과적으로 TFLite 출력은 다음 형태가 되도록 했다.

```text
input:  [1, 30, 332], float32
output: [1, 7], float32 probabilities
```

float32와 float16 두 모델을 모두 생성했다.

```text
best_sign_model_1_float32.tflite
best_sign_model_1_float16.tflite
```

float16은 모델 파일 크기를 줄이는 용도이며, 입력 ByteBuffer는 여전히 float32 기준으로 구성한다.

## TFLite 테스트 코드

PC에서 TFLite 모델을 직접 테스트하기 위해 실시간 TFLite 추론 코드를 만들었다. 기존 PyTorch 실시간 추론 코드와 같은 MediaPipe 전처리를 사용하고, TFLite Interpreter로 모델을 실행하도록 했다.

초기 실행 과정에서 TensorFlow, protobuf, JAX, `ml_dtypes` 관련 충돌이 있었다. 최종적으로 TensorFlow 2.15 계열에서 MediaPipe와 함께 동작하도록 환경을 맞추는 방향으로 정리했다.

## 모바일 전달 문서

모바일 담당 팀원에게 전달할 문서를 작성했다.

```text

```

문서에는 다음 내용을 포함했다.

```text
모델 파일 목록
label map
입력 shape
출력 shape
332차원 feature 구조
MediaPipe Holistic landmark 선택 방식
```

모바일에서는 `holistic_landmarker.task`로 landmark를 추출한 뒤, Python 학습/추론에서 사용한 동일 landmark만 골라 332차원 feature를 만들어 TFLite에 넣는 방식이 가능하다고 판단했다.

## 문장 시퀀스 논의

단어 인식 이후에는 gloss 형태의 단어 리스트를 한국어 문장으로 변환해야 했다. 다만 gloss에서 한국어 문장 변환은 팀원이 Gemma 3로 꽤 잘 해결했다고 하여, 우리는 모델이 예측한 단어들을 어디서부터 어디까지 하나의 문장으로 볼지에 집중하기로 했다.

문장 시작은 첫 번째 단어가 확정되는 순간으로 보았다. 단어 사이 준비 동작이나 손 위치 조정은 문장 시작으로 보지 않았다.

문장 끝은 양손이 모두 화면에서 사라진 상태가 1.2~1.5초 이상 지속될 때로 정했다. 한 손만 사라진 경우는 문장 끝으로 보지 않았다.

## 중복 제거 아이디어

연속 단어 시퀀스에서는 다음 후처리가 필요하다고 정리했다.

```text
1. 연속 중복 단어 제거
2. A-B-A 형태의 짧은 노이즈 제거
3. confidence와 margin 기준을 통과한 단어만 확정
4. 안정적으로 여러 번 등장한 단어만 sentenceBuffer에 추가
```

예시는 다음과 같다.

```text
raw:     [가다, 가다, 기차, 가다, 가다]
cleaned: [가다]
```

## 모델 선택 및 API 논의

gloss를 자연스러운 한국어 문장으로 바꾸는 LLM/SLLM 후보도 논의했다. NAVER CLOVA Speech는 STT/TTS 성격이어서 gloss 변환에는 맞지 않는다고 판단했다.

SSAFY GMS 크레딧을 활용할 수 있다면 GPT 계열 소형 모델을 API로 사용하는 방안도 고려했다. 다만 이후 팀원이 Gemma 3 기반 변환을 잘 구현했다고 하여, 우리 쪽 우선순위는 문장 구간 분리와 단어 시퀀스 안정화로 조정했다.
