## 롤백한 실험
- history reducer 기반 후처리
- hold boundary 기반 단어 경계 분리
- committed words 누적
- hold 이후 동적 minimum prediction frames 조정

롤백 이유:
- 2단어 일부 케이스에서는 가능성이 있었지만, 3단어 이상 연속 문장에서는 일관성이 부족했다.
- hold-only 방식은 사용자가 기대한 자연스러운 문장 분리 UX를 안정적으로 만들지 못했다.
- 실험 로직을 계속 누적하면 원인 분리가 어려워져, 기준선을 다시 단순화하는 편이 더 유리했다.

## Python 추론 코드 비교 결과
- 비교 대상:
  - `docs/Ondevice_v1/inference_realtime_tflite_v6.py`
  - `mobile/app/src/main/java/com/ssafy/mobile/core/vision/feature/LandmarkFeatureEncoder.kt`
  - `mobile/app/src/main/java/com/ssafy/mobile/core/vision/SignPredictionStabilizer.kt`

### 확인한 점
- 입력 차원과 landmark 선택 순서는 Python과 Android가 크게 다르지 않았다.
- `normalize_mode`는 Python 쪽 V6 설정 기준 `none`이었고, Android도 `RAW`를 사용하고 있어 정규화 모드 자체는 핵심 차이로 보기 어려웠다.
- 가장 큰 차이는 Android가 기본적으로 손 landmark를 forward-fill 하던 점이었다.

### 해석
- Python은 손이 없으면 해당 손을 0으로 채운다.
- Android는 이전 손 landmark를 재사용할 수 있었고, 이 차이가 연속 단어 전환 구간에서 이전 단어 손 모양을 더 오래 남게 만들었을 가능성이 높았다.
- 실제로 `forward-fill off` 적용 후 결과가 크게 개선되었다.

- `기차 -> 놀다 -> 주다`: 최종 문장 `기차 놀다 주다`
- `기차 -> 놀다 -> 돕다`: 최종 문장 `기차 놀다 돕다`
- `엄마 -> 좋다`: 최종 문장 `엄마 좋다`
